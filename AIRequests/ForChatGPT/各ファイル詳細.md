# 各ファイル詳細

対象: `build_windows.ps1`、`backend/`、`frontend/` 配下のうち、`.gitignore` に含まれず、`__init__.py` と `*.md` ではないファイル。

## ルート

### `build_windows.ps1`
- 概要: Windows 向け配布実行ファイルを作成する PowerShell スクリプト。Python 仮想環境、依存関係、フロントエンドビルド、PyInstaller、同梱ランタイム検証、portable zip 作成を扱う。
- 依存関係: `backend/requirements.txt` と `pyinstaller==6.20.0` をビルド用 venv にインストールする。
- PyInstaller 収集: `webview`、`pythonnet`、`clr_loader`、`cffi`、`PIL`、`send2trash` を `--collect-all` で明示収集する。
- 主な関数:
  - `Invoke-CheckedCommand`: 外部コマンドを実行し、失敗時に例外化する。
  - `Remove-DirectoryIfExists`: 既存ディレクトリをリトライ付きで削除する。
  - `Test-PythonExecutable`: Python 実行ファイルとして利用できるか検証する。
  - `Test-VenvUsesBootstrapPython`: venv が想定 Python で作られているか確認する。
  - `Assert-BuildPythonRuntime`: ビルド Python の 64bit と pythonnet import を検証する。
  - `Assert-PythonRuntimeBundled`, `Assert-PythonNetBundled`, `Assert-WindowsGuiRuntimeBundled`: 配布物に必須ランタイムが含まれることを検証する。

## Backend

### `backend/main.py`
- 概要: pywebview アプリケーションのエントリーポイント。起動ログ、ランタイム検証、表示 URL 解決、ドラッグアンドドロップ登録、Vue へのイベント送信、ウィンドウ生成を担当する。
- 主な要素:
  - `TeeStream`: 標準出力/標準エラーを元ストリームとログファイルへ複写する。
  - `candidate_log_dirs()`, `open_startup_log()`, `configure_startup_logging()`: 起動ログ出力先とログ複写を設定する。
  - `smoke_test_runtime()`: 配布ランタイムの簡易検証を行う。
  - `resolve_entry_url()`: 開発サーバーまたはビルド済み `frontend/dist` の URL を返す。
  - `create_drop_items(event)`, `register_handlers(window, api)`: DOM drop イベントを画像登録 API へ接続する。
  - `dispatch_frontend_event(...)`, `create_import_event_detail(...)`: 登録進捗を Vue 側 `CustomEvent` として通知する。

### `backend/api/api_response.py`
- 概要: フロントエンドへ返す API レスポンスの共通 DTO。
- 主な要素:
  - `ApiResponse`: `success`, `message`, `data` を保持する。
  - `to_dict()`: pywebview 経由で返す辞書へ変換する。

### `backend/api/app_api.py`
- 概要: pywebview から Vue に公開する統合 API。各専門 API への薄い委譲層。
- 主な責務:
  - DB ライフサイクル、基本情報、画像登録、画像カタログ、OS 操作 API を組み立てる。
  - 画像検索、詳細更新、一括属性更新、管理対象除外、ごみ箱移動、リネーム、ファイル移動、タグ/caption/ワイルドカード出力、マスタメンテナンス、メタ情報取得を公開する。
- 削除系の現状:
  - `remove_image_files_from_catalog(payload)`: 実画像ファイルを残して AZViewer 管理対象から除外する。
  - `move_image_files_to_trash(payload)`: 実画像ファイルを OS ごみ箱へ移動し、成功分を管理対象から除外する。
  - 即時完全削除 API は公開していない。
- 追加操作:
  - `rename_image_file(payload)`: 同一フォルダ内でファイル名を変更する。

### `backend/api/app_lifecycle_api.py`
- 概要: アプリ起動時の DB 初期化、重複タグ構成用 `tag_hash` 補完、登録済みファイル存在確認、不足サムネイル生成、初期検索結果取得を担当する。
- 主な要素:
  - `bootstrap_app()`: 起動時整理、`tag_hash` 補完、サムネイル補完、初期検索結果取得をまとめて実行する。
  - `_run_startup_cleanup()`: 実ファイルが存在しない登録情報を整理する。
  - `_ensure_tag_hashes()`: `tag_hash` が空の場合に再構築する。
  - `_ensure_thumbnail_cache()`: 初期検索結果のサムネイルを補完する。

### `backend/api/database_lifecycle_manager.py`
- 概要: アプリ全体で共有する SQLite 接続のライフサイクルを管理する。
- 主な要素:
  - `initialize()`: `ConnectionManager` を通じて DB 接続を初期化する。
  - `get_connection()`: 初期化済み接続を返す。
  - `close()`: 接続を閉じる。

### `backend/api/default_template_api.py`
- 概要: アプリ名、バージョン、説明、ヘルスチェックなどの基本 API を提供する。
- 主な要素:
  - `initialize_app()`, `get_app_info()`, `health_check()`: Vue 側起動確認と表示用情報を返す。

### `backend/api/image_catalog_api.py`
- 概要: 画像カタログ操作の中心 API。検索、詳細更新、一括属性更新、管理対象除外、ごみ箱移動、リネーム、ファイル移動、タグ/caption/ワイルドカード入出力、候補取得、マスタメンテナンス、画像/メタ情報取得を担当する。
- 主な公開 API:
  - `search_image_files(payload)`: 条件付き画像検索とページング。
  - `update_image_file_detail(payload)`: レーティング、チェック、お気に入り、コメント、タグ、モデルを同一トランザクションで更新。
  - `bulk_update_image_file_attributes(payload)`: 選択画像の属性を一括更新。
  - `remove_image_files_from_catalog(payload)`: DB レコード、タグリンク、モデルリンク、`tag_hash`、サムネイルキャッシュを削除し、実画像ファイルは残す。
  - `move_image_files_to_trash(payload)`: `send2trash` で実画像ファイルをごみ箱へ移動し、成功分または実ファイル不存在分のみ管理対象から除外する。
  - `rename_image_file(payload)`: ID から現行パスを再取得し、同一フォルダ内でファイル名を変更する。拡張子変更、上書き、パス区切り文字、Windows 禁止文字を拒否する。
  - `move_image_files_to_folder(payload)`: 実画像ファイルを指定フォルダへ移動し、成功分の DB パスを更新する。
  - `export_selected_image_tags(payload)`: 選択画像のタグを画像と同名の `.txt` として出力する。
  - `fetchLocalImage(payload)`, `fetchLocalImageThumb(payload)`, `fetch_image_metadata(payload)`: 元画像、サムネイル、メタ情報を取得する。
  - `fetch_tags_for_search`, `fetch_folders_for_search`, `fetch_models_for_search`: 検索候補を返す。
  - `fetch_tags_for_maintenance`, `delete_tag_master`, `replace_tag_master`, `delete_unused_tags`: タグマスタメンテナンス。
  - `fetch_models_for_maintenance`, `delete_model_master`, `replace_model_master`, `delete_unused_models`: モデルマスタメンテナンス。
  - `fetch_duplicate_tag_sets`, `import_prompt_tags`, `import_caption_tags`, `bulk_add_tags`, `export_wildcard_text`: 重複タグ構成検索、タグ読み込み、一括タグ追加、ワイルドカード出力。
  - `open_containing_folder(payload)`: 対象画像の親フォルダを OS で開く。
- ファイル操作方針:
  - 実画像ファイルの即時完全削除は行わない。
  - 実画像ファイルの除去は `send2trash` によるごみ箱移動のみ。
  - サムネイルキャッシュ削除では `ThumbnailCacheService` が `unlink()` を使う。
  - リネームの DB 更新失敗時は、可能な場合に実ファイル名を元へ戻す。

### `backend/api/image_register_api.py`
- 概要: ファイル選択、フォルダ選択、ドラッグアンドドロップ由来の画像登録 API。
- 主な要素:
  - `import_selected_items(payload)`: 選択またはドロップされたファイル/フォルダを登録する。
  - `select_files_dialog()`, `select_folder_dialog()`: pywebview のネイティブ選択ダイアログを開く。
  - 登録後はサムネイル生成も行う。

### `backend/api/os_operation_api.py`
- 概要: OS 操作 API。
- 主な要素:
  - `open_containing_folder(payload)`: 指定画像の親フォルダを開く。

### `backend/db/connection_manager.py`
- 概要: SQLite 接続作成と DB ファイル配置を担当する。
- 主な要素:
  - `connect()`: `data/azviewer.db` への接続を作成し、row factory や foreign key などを設定する。
  - `close()`: 接続を閉じる。

### `backend/models/image_file.py`
- 概要: バックエンド API とリポジトリで使う DTO 群。
- 主な DTO:
  - `ImageFileRecord`, `ImageFileListItem`: 登録用レコードと一覧表示用データ。
  - `SearchImageFilesResult`: 検索結果。
  - `BulkAttributeUpdateResult`: 一括属性更新結果。
  - `CatalogRemovalFailure`, `CatalogRemovalResult`: 管理対象除外結果。
  - `TrashMoveFailure`, `TrashMoveResult`: ごみ箱移動結果。
  - `ImageRenameResult`: リネーム結果。
  - `FileMoveFailure`, `FileMoveResult`: ファイル移動結果。
  - `TagCaptionExport*`, `CaptionTagImport*`, `PromptTagImport*`, `BulkTagAdd*`: タグ/caption/プロンプト系処理結果。
  - `MasterMaintenanceItem`, `MasterMaintenanceSearchResult`, `MasterDeleteResult`, `MasterReplaceResult`, `MasterBulkDeleteResult`: マスタメンテナンス結果。
  - `DuplicateTagSetItem`, `WildcardExportResult`, `ImportResult`: 重複タグ構成、ワイルドカード出力、画像登録結果。

### `backend/repositories/image_file_repository.py`
- 概要: SQLite の画像、タグ、モデル、`tag_hash` テーブルへのアクセスを担当する。
- 主な責務:
  - テーブル作成、画像登録、検索、詳細更新、属性一括更新、ファイルパス更新、ファイル名/パス/フォルダ更新。
  - タグリンク、モデルリンク、`tag_hash` の同期。
  - タグ/モデルの検索候補取得、マスタ削除・置換・未使用削除。
  - 重複タグ構成検索。
- 重要メソッド:
  - `find_by_ids(record_ids)`, `find_by_paths(paths)`, `find_ids_by_paths(paths)`: 登録情報取得。
  - `search(...)`: 絞り込み、ページング、ソート付き検索。
  - `remove_from_catalog_by_ids(record_ids)`: `tag_image_link`、`image_model_to_file_data`、`tag_hash`、`image_file_data` から対象を 1 トランザクションで削除する。
  - `update_paths(updates)`: ファイル移動後の `path` / `folder` を更新する。
  - `update_file_identity(record_id, filename, path, folder)`: リネーム後の `filename` / `path` / `folder` を更新する。
  - `update_detail(...)`: 画像詳細を更新する。
  - `replace_tags(...)`, `replace_model(...)`, `replace_tag_hash(...)`: タグ・モデル・派生インデックスを更新する。

- 概要: 画像ファイルのメタ情報テキスト取得を担当する。

  - PNG info、EXIF などから表示用メタ情報を組み立てる。

- 概要: Stable Diffusion WebUI 系メタ情報から prompt タグと生成元モデルを一括読み取りする。
- 主な責務:
  - Positive prompt の抽出。
  - タグ正規化と登録。
  - モデル名抽出とリンク。
  - タグ処理とモデル処理の件数を集計する。

### `backend/services/startup_cleanup_service.py`
- 概要: 起動時の登録情報整合性整理を担当する。
- 主な責務:
  - 実ファイルが存在しない登録済み画像を検出する。
  - DB 登録情報とサムネイルキャッシュを削除する。

### `backend/services/tag_caption_export_service.py`
- 概要: 選択画像のタグを画像と同名の caption `.txt` へ出力する。
- 主な責務:
  - タグ未設定画像のスキップ。
  - 既存 `.txt` の上書き。
  - 成功、スキップ、失敗件数を集計する。

### `backend/services/tag_normalize_service.py`
- 概要: タグ文字列の正規化を担当する。
- 主な責務:
  - 区切り文字処理、空要素除去、重複排除。
  - Stable Diffusion prompt 由来の括弧や重み表現の整理。

### `backend/services/thumbnail_cache_service.py`
- 概要: サムネイルキャッシュの生成、取得、削除を担当する。
- 主な責務:
  - `ensure_thumbnails(items, missing_only=True)`: 不足サムネイルを生成する。
  - `create_thumbnail(record_id, source_path)`: Pillow で 300px サムネイル PNG を生成する。
  - `delete_thumbnail(record_id)`, `delete_thumbnails(record_ids)`: サムネイルキャッシュを削除する。
  - `get_thumbnail_bytes(record_id)`: フロント表示用にバイト列を返す。

### `backend/services/wildcard_export_service.py`
- 概要: Stable Diffusion WebUI ワイルドカード向けテキスト出力を担当する。
- 主な責務:
  - 保存先ダイアログ。
  - 新規保存または追記。
  - 出力行数とキャンセル状態を返す。

### `frontend/src/App.vue`
- 概要: Vue アプリ全体の状態連携とイベントハンドリングの中心。
- 主な責務:
  - 起動、検索、選択状態、詳細モーダル、各種モーダル、toast、LoadingOverlay を接続する。

  - ファイル操作: ファイルを移動、ごみ箱へ移動、選択画像を管理対象から除外。
  - タグ操作: プロンプト読み取り、caption タグ読み込み、一括タグ追加、重複タグ構成検索。
  - マスタメンテナンス: タグ/モデル。

### `frontend/src/components/tile/ImageDetailModal.vue`
- 概要: 画像詳細表示モーダル。API 呼び出しや再検索は持たず、表示と操作イベント通知に限定する。
- 主な機能:
  - 詳細画像を通常表示/実寸表示で切り替える。
  - 現在ページ内の前へ/次へ移動ボタンを表示する。
  - ArrowLeft / ArrowRight / Escape を処理する。
  - リネームモーダル表示中、入力要素フォーカス中、contenteditable、IME 入力中は左右キー移動を無視する。
  - `move-to-trash`, `rename`, `edit-current`, `previous`, `next`, `close` を emit する。
- 表示内容:
  - タイトルは `filename`。
  - 補助表示として `{currentIndex + 1} / {totalCount}` を表示する。

### `frontend/src/components/tile/ImageMetadataModal.vue`

### `frontend/src/components/tile/ImageModelSelectModal.vue`

### `frontend/src/components/tile/tile.vue`
- 概要: 画像タイルのコンテナ。表示/編集切り替え、選択チェック、詳細表示、メタ情報表示を扱う。
- 主な責務:
  - タイルルートに `image-tile-{id}` を付与し、詳細モーダルの属性編集ショートカットからスクロールできるようにする。
  - `editingImageId` が自身の ID と一致したら edit モードに入る。
  - 保存またはキャンセル時に `edit-finished` を emit する。
  - 単体削除、管理対象除外、ごみ箱移動の処理は持たない。

### `frontend/src/composables/useImageDetail.js`
- 概要: 画像詳細モーダルの状態管理。
- 主な状態:
  - `detailModal.show`, `item`, `imageSrc`, `items`, `currentIndex`, `isLoading`。
- 主な機能:
  - `openDetail({ item, items })`: 現在ページ内リストと対象画像で詳細を開く。
  - `loadDetailImage(item)`: 元画像を読み込む。
  - `hasPrevious`, `hasNext`: 現在ページ内の前後移動可否。
  - `movePrevious()`, `moveNext()`: 現在ページ内で前後移動する。
  - `refreshDetailItemsAfterSearch(items, preferredId, fallbackId)`: 再検索後の詳細対象を更新し、候補がなければ閉じる。

### `frontend/src/composables/useImageMetadata.js`

### `frontend/src/composables/useImageMutations.js`
- 概要: 画像に対する更新系操作をまとめる composable。
