# 各ファイル詳細

対象: `build_windows.ps1`、`backend/`、`frontend/` 配下のうち、`.gitignore` に含まれず、`__init__.py` と `*.md` ではないファイル。

## ルート

### `build_windows.ps1`
- 概要: Windows 向け配布実行ファイルを作成する PowerShell スクリプト。Python 仮想環境、依存関係、フロントエンドビルド、PyInstaller、同梱ランタイム検証を扱う。
- 主な関数:
  - `Invoke-CheckedCommand`: 外部コマンドを実行し、失敗時に例外化する。
  - `Remove-DirectoryIfExists`: 既存ディレクトリを削除する。
  - `Test-PythonExecutable`: Python 実行ファイルとして利用できるか検証する。
  - `Test-VenvUsesBootstrapPython`: venv が想定 Python で作られているか確認する。
  - `Assert-BuildPythonRuntime`: ビルドに使う Python ランタイムを検証する。
  - `Assert-PathExists`: 必須パスの存在を検証する。
  - `Assert-BundledFile`: 配布物に指定ファイルが含まれることを検証する。
  - `Assert-PythonRuntimeBundled`: Python ランタイム同梱を検証する。
  - `Assert-PythonNetBundled`: pythonnet 関連ファイル同梱を検証する。
  - `Assert-WindowsGuiRuntimeBundled`: Windows GUI 実行に必要なランタイム同梱を検証する。

## Backend

### `backend/main.py`
- 概要: pywebview アプリケーションのエントリーポイント。起動ログ、ランタイム検証、表示 URL 解決、ドラッグアンドドロップ登録、フロントエンドイベント送信、ウィンドウ生成を担当する。
- 主なクラス/関数:
  - `TeeStream`: 標準出力/標準エラーを元ストリームとログファイルへ複写する。
  - `candidate_log_dirs()`: 起動ログ出力先候補を返す。
  - `open_startup_log()`: 起動ログファイルを開く。
  - `configure_startup_logging()`: 標準出力/標準エラーのログ複写を設定する。
  - `run_with_startup_logging()`: ログ設定付きで `main()` を実行する。
  - `smoke_test_runtime()`: 配布ランタイムの簡易検証を行う。
  - `resolve_entry_url()`: 開発サーバー URL またはビルド済みフロントエンド HTML の URL を返す。
  - `create_drop_items(event)`: drop イベントから登録用 item 配列を作る。
  - `dispatch_frontend_event(window, event_name, detail)`: Vue 側へ `CustomEvent` を送る。
  - `create_import_event_detail(result)`: 登録 API 結果をイベント detail へ変換する。
  - `register_handlers(window, api)`: DOM drop イベントに画像登録処理を紐づける。
  - `main()`: pywebview ウィンドウを作成し、アプリケーションを起動する。
