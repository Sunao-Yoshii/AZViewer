# 各ファイル詳細

対象: `AZViewer.spec`、`build_windows.ps1`、`backend/`、`frontend/` 配下のうち、`.gitignore` に含まれず、`__init__.py` と `*.md` ではないファイル。

## ルート

### `AZViewer.spec`
- 概要: PyInstaller 用 spec。`backend/main.py` をエントリーポイントにし、`frontend/dist` を同梱する。
- 収集対象: `webview`、`pythonnet`、`clr_loader`、`cffi`、`PIL`、`send2trash`、`sqlalchemy`、`greenlet`。
- 備考: ごみ箱移動機能で必要な `send2trash` と DB 接続で必要な `sqlalchemy` / `greenlet` は spec 側でも明示収集している。

### `build_windows.ps1`
- 概要: Windows 向け配布実行ファイルと portable zip を作成する PowerShell スクリプト。
- 主な処理: Python venv 作成、`backend/requirements.txt` と `pyinstaller==6.20.0` の導入、`npm run build`、PyInstaller 実行、同梱ランタイム検証、zip 作成。
- PyInstaller 収集: `webview`、`pythonnet`、`clr_loader`、`cffi`、`PIL`、`send2trash`、`sqlalchemy`、`greenlet` を `--collect-all` で明示収集する。
- 主な関数: `Invoke-CheckedCommand`、`Remove-DirectoryIfExists`、`Test-PythonExecutable`、`Test-VenvUsesBootstrapPython`、`Assert-BuildPythonRuntime`、`Assert-PythonRuntimeBundled`、`Assert-PythonNetBundled`、`Assert-WindowsGuiRuntimeBundled`。

## Backend

### `backend/main.py`
- 概要: pywebview アプリケーションのエントリーポイント。
- 主な責務: 起動ログ設定、配布ランタイム検証、開発サーバーまたは `frontend/dist` の表示 URL 解決、ウィンドウ生成、ドラッグアンドドロップ登録、Vue への `CustomEvent` 通知。
- 主な要素: `TeeStream`、`candidate_log_dirs()`、`open_startup_log()`、`configure_startup_logging()`、`smoke_test_runtime()`、`resolve_entry_url()`、`create_drop_items()`、`register_handlers()`、`dispatch_frontend_event()`。

### `backend/api/api_response.py`
- 概要: pywebview 経由で返す API レスポンス共通 DTO。
- 主な要素: `ApiResponse(success, message, data)` と `to_dict()`。

### `backend/api/app_api.py`
- 概要: Vue へ公開する統合 API。専門 API への薄い委譲層。
- 主な公開範囲: 起動処理、アプリ情報、画像登録、画像検索、詳細更新、一括属性更新、タグ/モデル候補、マスタメンテナンス、タグ/caption/ワイルドカード入出力、重複タグ構成検索、画像/サムネイル/メタ情報取得、OS 操作。
- ファイル操作 API:
  - `remove_image_files_from_catalog(payload)`: 実画像を残して管理対象から除外する。
  - `move_image_files_to_trash(payload)`: 実画像を OS ごみ箱へ移動し、成功分を管理対象から除外する。
  - `rename_image_file(payload)`: 同一フォルダ内で画像ファイル名を変更する。
  - `move_image_files_to_folder(payload)`: 実画像を指定フォルダへ移動し、DB パスを更新する。
  - `open_containing_folder(payload)`: 対象画像の親フォルダをエクスプローラで開く。
- 備考: 旧来の即時完全削除 API は公開していない。

### `backend/api/app_lifecycle_api.py`
- 概要: アプリ起動時の初期化処理をまとめる API。
- 主な処理: DB 初期化、実ファイル不存在レコードの整理、`tag_hash` 補完、サムネイル補完、初期検索結果取得。
- 主な要素: `bootstrap_app()`、`_run_startup_cleanup()`、`_ensure_tag_hashes()`、`_ensure_thumbnail_cache()`。

### `backend/api/database_lifecycle_manager.py`
- 概要: アプリ全体で共有する SQLAlchemy Core `Engine` のライフサイクル管理。
- 主な要素: `initialize()`、`get_engine()`、`dispose()`、互換用 `close()`。
- 備考: 従来の単一接続共有は行わない。

### `backend/api/default_template_api.py`
- 概要: 基本情報とヘルスチェック API。
- 主な要素: `initialize_app()`、`get_app_info()`、`health_check()`。

### `backend/api/image_catalog_api.py`
- 概要: 画像カタログ操作の中心 API。
- 主な公開 API:
  - `search_image_files(payload)`: 条件付き検索、ページング、ソート。
  - `update_image_file_detail(payload)`: レーティング、チェック、お気に入り、コメント、タグ、モデルを更新する。
  - `bulk_update_image_file_attributes(payload)`: 選択画像の属性を一括更新する。
  - `remove_image_files_from_catalog(payload)`: DB レコード、関連リンク、`tag_hash`、サムネイルキャッシュを削除し、実画像は残す。
  - `move_image_files_to_trash(payload)`: `send2trash` で実画像をごみ箱へ移動し、成功分または実ファイル不存在分を管理対象から除外する。
  - `rename_image_file(payload)`: ID から現行パスを取得し、同一フォルダ内でファイル名を変更する。
  - `move_image_files_to_folder(payload)`: 実画像を指定フォルダへ移動し、成功分の DB パスを更新する。
  - `export_selected_image_tags(payload)`: 選択画像のタグを画像と同名の `.txt` へ出力する。
  - `fetchLocalImage(payload)`: 元画像を base64 data URL として返す。
  - `fetchLocalImageThumb(payload)`: サムネイルを base64 data URL として返す。
  - `fetch_image_metadata(payload)`: PNG info / EXIF などのメタ情報を返す。
  - `fetch_tags_for_search`、`fetch_folders_for_search`、`fetch_models_for_search`: 検索候補を返す。
  - `fetch_tags_for_maintenance`、`delete_tag_master`、`replace_tag_master`、`delete_unused_tags`: タグマスタメンテナンス。
  - `fetch_models_for_maintenance`、`delete_model_master`、`replace_model_master`、`delete_unused_models`: モデルマスタメンテナンス。
  - `fetch_duplicate_tag_sets`、`import_prompt_tags`、`import_caption_tags`、`bulk_add_tags`、`export_wildcard_text`: タグ関連の追加機能。
- リネーム検証: 空文字、パス区切り文字、Windows 禁止文字、拡張子変更、既存ファイル上書きを拒否する。DB 更新失敗時は可能な範囲で実ファイル名を元に戻す。
- ファイル削除方針: 実画像の即時完全削除は行わず、実画像の除去はごみ箱移動のみ。サムネイルキャッシュ削除では `ThumbnailCacheService` がキャッシュファイルを削除する。

### `backend/api/image_register_api.py`
- 概要: ファイル選択、フォルダ選択、ドラッグアンドドロップ由来の画像登録 API。
- 主な要素: `import_selected_items(payload)`、`select_files_dialog()`、`select_folder_dialog()`。
- 備考: 登録処理後にサムネイル生成も行う。

### `backend/api/os_operation_api.py`
- 概要: OS 連携 API。
- 主な要素: `open_containing_folder(payload)`。指定された画像パスの親フォルダをエクスプローラで開く。

### `backend/db/database_engine.py`
- 概要: AZViewer 用 SQLite SQLAlchemy `Engine` を作成する。
- 主な設定: `NullPool`、`PRAGMA foreign_keys = ON`、`PRAGMA busy_timeout = 30000`、`PRAGMA journal_mode = WAL`。
- 備考: DB ファイルは既定で `data/az_data.sqlite3` を利用する。

### `backend/models/image_file.py`
- 概要: バックエンド API とリポジトリで使う DTO 群。
- 主な DTO: `ImageFileRecord`、`ImageFileListItem`、`SearchImageFilesResult`、`BulkAttributeUpdateResult`、`CatalogRemovalFailure`、`CatalogRemovalResult`、`TrashMoveFailure`、`TrashMoveResult`、`ImageRenameResult`、`FileMoveFailure`、`FileMoveResult`、`TagCaptionExport*`、`CaptionTagImport*`、`PromptTagImport*`、`BulkTagAdd*`、`MasterMaintenance*`、`DuplicateTagSetItem`、`WildcardExportResult`、`ImportResult`。

### `backend/repositories/image_file_repository.py`
- 概要: SQLAlchemy Core `Engine` 経由で SQLite の画像、タグ、モデル、`tag_hash` テーブルアクセスを担当する。
- 主な責務: テーブル作成、画像登録、検索、詳細更新、一括属性更新、管理対象除外、ファイル移動後パス更新、リネーム後ファイル識別情報更新、タグ/モデルリンク同期、マスタメンテナンス、重複タグ構成検索。
- 重要メソッド: `find_by_ids()`、`find_by_paths()`、`find_ids_by_paths()`、`search()`、`remove_from_catalog_by_ids()`、`update_paths()`、`update_file_identity()`、`update_detail()`、`replace_tags()`、`replace_model()`、`replace_tag_hash()`。
- 備考: 旧来の単体/複数即時削除用リポジトリメソッドは存在しない。

### `backend/requirements.txt`
- 概要: バックエンド実行・配布に必要な Python 依存関係。
- 主な依存: `pywebview`、`pythonnet`、`clr-loader`、`Pillow`、`send2trash`。
- DB 接続: `SQLAlchemy>=2.0,<3.0`。

### `backend/services/bulk_tag_add_service.py`
- 概要: 選択画像へのタグ一括追加。
- 主な責務: 入力タグ正規化、既存タグとの統合、タグリンクと `tag_hash` 更新、件数集計。

### `backend/services/caption_tag_import_service.py`
- 概要: 選択画像と同名の caption `.txt` を読み込み、タグとして追加登録する。
- 主な責務: caption ファイル探索、タグ正規化、既存タグ保持、更新/スキップ/失敗件数集計。

### `backend/services/dialog_service.py`
- 概要: pywebview のファイル/フォルダ選択、保存ダイアログを扱うサービス。
- 主な責務: アクティブウィンドウ取得、ネイティブダイアログ表示、返却値整形。

### `backend/services/file_scan_service.py`
- 概要: ファイル/フォルダ入力から登録対象画像ファイルを抽出する。
- 主な責務: 対応拡張子判定、フォルダ直下走査、登録対象パスの重複排除。

### `backend/services/image_file_import_service.py`
- 概要: 画像ファイル登録の業務処理。
- 主な責務: 既存登録済みパス除外、`ImageFileRecord` 作成、DB 登録、サムネイル生成、登録/スキップ/失敗件数返却。

### `backend/services/image_metadata_service.py`
- 概要: 画像ファイルのメタ情報テキスト取得。
- 主な責務: Pillow で画像を開き、PNG info / EXIF などから表示用メタ情報を組み立てる。

### `backend/services/prompt_tag_import_service.py`
- 概要: Stable Diffusion WebUI 系メタ情報から prompt タグと生成元モデルを一括読み取りする。
- 主な責務: Positive prompt 抽出、タグ正規化、モデル名抽出、タグ/モデル登録、件数集計。

### `backend/services/startup_cleanup_service.py`
- 概要: 起動時の登録情報整合性整理。
- 主な責務: 実ファイル不存在の登録済み画像を検出し、DB 登録情報とサムネイルキャッシュを削除する。

### `backend/services/tag_caption_export_service.py`
- 概要: 選択画像のタグを画像と同名の caption `.txt` へ出力する。
- 主な責務: タグ未設定画像スキップ、既存 `.txt` 上書き、成功/スキップ/失敗件数集計。

### `backend/services/tag_normalize_service.py`
- 概要: タグ文字列の正規化。
- 主な責務: 区切り文字処理、空要素除去、重複排除、Stable Diffusion prompt 由来の括弧や重み表現の整理。

### `backend/services/thumbnail_cache_service.py`
- 概要: サムネイルキャッシュの生成、取得、削除。
- 主な要素: `ensure_thumbnails()`、`create_thumbnail()`、`delete_thumbnail()`、`delete_thumbnails()`、`get_thumbnail_bytes()`。

### `backend/services/wildcard_export_service.py`
- 概要: Stable Diffusion WebUI ワイルドカード向けテキスト出力。
- 主な責務: 保存先ダイアログ、新規保存/追記、出力行数とキャンセル状態返却。

### `frontend/src/App.vue`
- 概要: Vue アプリ全体の状態連携とイベントハンドリングの中心。
- 主な責務: 起動処理、検索、選択状態、詳細ビューア、リネームモーダル、各種モーダル、toast、LoadingOverlay を接続する。
- 詳細ビューア連携:
  - `ImageDetailViewer` へ現在ページの画像リスト、表示中 item、画像 data URL、前後移動可否、処理中状態を渡す。
  - 前後移動は `useImageDetail.movePrevious()` / `moveNext()` に委譲する。
  - `move-to-trash` は確認後に `useImageMutations.moveSingleImageToTrash()` を呼び、再検索後に次候補、前候補、閉じるの順で表示を更新する。
  - `rename` は `ImageRenameModal` を開き、保存時に `useImageMutations.renameSingleImageFile()` を呼ぶ。
  - `edit-current` は詳細ビューアを閉じ、対象タイルへスクロールして edit モードへ切り替える。
  - `open-containing-folder` は `useImageMutations.openContainingFolder({ path })` を呼ぶ。
- 備考: `editingImageId` により詳細ビューアから一覧タイル編集へ遷移できる。

### `frontend/src/components/tile/ImageDetailViewer.vue`
- 概要: 画像詳細ビューア。API 呼び出しや再検索は持たず、表示と操作イベント通知に限定する。
- Props: `show`、`item`、`imageSrc`、`currentIndex`、`totalCount`、`hasPrevious`、`hasNext`、`isLoading`、`isBusy`。
- 主な emit: `close`、`previous`、`next`、`move-to-trash`、`rename`、`edit-current`、`open-containing-folder`。
- 表示機能:
  - 画像表示領域とサイドバー操作領域で構成する。
  - 通常表示では縦長/横長画像を表示領域内に収める。
  - 実寸表示では画像を縮小せず、必要に応じてスクロールできる。
  - タイトル、フルパス、現在位置、レーティング、チェック、お気に入り、モデル、タグ、コメントを表示する。
- 操作:
  - 「エクスプローラで開く」ボタンで親フォルダを開くイベントを emit する。
  - 「前へ」「次へ」で現在ページ内を移動する。
  - 「実寸表示」「画面に合わせる」を切り替える。
  - 「属性編集」で対象タイルの edit モードへ遷移するイベントを emit する。
  - 「リネーム」で `ImageRenameModal` を開くイベントを emit する。
  - 「ごみ箱へ移動」で単体ごみ箱移動イベントを emit する。
  - 「閉じる」でビューアを閉じる。
- キーボード:
  - `ArrowLeft`: 前の画像へ移動。
  - `ArrowRight`: 次の画像へ移動。
  - `Escape`: 閉じる。
  - `Delete`: 表示中画像を「ごみ箱へ移動」するイベントを emit する。
  - 入力要素フォーカス中、contenteditable、IME 入力中、リネームモーダル表示中はビューアのショートカットを無視する。

### `frontend/src/components/tile/ImageMetadataModal.vue`
- 概要: 画像メタ情報表示モーダル。
- 主な機能: メタ情報取得結果表示、コピー、タグ入力への流用。

### `frontend/src/components/tile/tile.vue`

- 主な責務:
  - タイルルートに `image-tile-{id}` を付与し、詳細ビューアの属性編集ショートカットからスクロールできるようにする。

### `frontend/src/composables/useDuplicateTagSetSearch.js`

### `frontend/src/composables/useImageDetail.js`
- 概要: 画像詳細ビューアの状態管理。
- 主な状態: `detailModal.show`、`item`、`imageSrc`、`items`、`currentIndex`、`isLoading`。
- 主な機能:
  - `openDetail({ item, items })`: 現在ページ内リストと対象画像で詳細ビューアを開く。
  - `loadDetailImage(item)`: 元画像を読み込む。
  - `hasPrevious`、`hasNext`: 現在ページ内の前後移動可否。
  - `movePrevious()`、`moveNext()`: 現在ページ内で前後移動する。
  - `refreshDetailItemsAfterSearch(items, preferredId, fallbackId)`: 再検索後の詳細対象を更新し、候補がなければ閉じる。

### `frontend/src/composables/useImageImport.js`

### `frontend/src/composables/useImageMetadata.js`
- 概要: 画像メタ情報モーダルの状態と API 呼び出しを管理する。
- 主な機能: メタ情報取得、コピー、タグ入力への流用。

### `frontend/src/composables/useImageMutations.js`
- 概要: 画像に対する更新系操作をまとめる composable。
- 主な機能:
  - 詳細保存 `handleSaveDetail(payload)`。
  - 一括属性更新、一括タグ追加、caption タグ読み込み、タグ caption 出力、ファイル移動。
  - 選択画像の管理対象除外。
  - 選択画像のごみ箱移動。
  - 詳細ビューア用の単体ごみ箱移動 `moveSingleImageToTrash({ id })`。
  - 詳細ビューア用の単体リネーム `renameSingleImageFile({ id, filename })`。
  - 詳細ビューア用の親フォルダ表示 `openContainingFolder({ path })`。
- 備考: `openContainingFolder` は成功時 toast を出さず、失敗時のみ通知する。

### `frontend/src/composables/useImageSearch.js`
