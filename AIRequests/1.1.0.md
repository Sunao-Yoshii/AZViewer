# 実装仕様書

## 表示中画像の一括選択・一括選択解除機能追加

## 1. 目的

画像一覧画面において、現在画面上に表示されているすべての画像を一括で選択、または一括で選択解除できるようにする。

本機能は、既存の以下の一括操作を利用しやすくするための補助機能である。

```text
- ワイルドカード出力
- ファイル移動
- 選択画像削除
```

既存仕様では、画像選択状態は DB に保存せず、`App.vue` 側の一時状態として保持する設計である。今回もこの方針を維持する。

## 2. 変更概要

`AppHeader.vue` に、現在表示中の画像をまとめて選択・解除するボタンを追加する。

```text
表示中をすべて選択
```

現在表示中の画像がすべて選択済みの場合は、同じボタンを以下の表示に切り替える。

```text
表示中の選択を解除
```

## 3. 基本方針

## 3.1 対象は「現在表示中」の画像のみ

本機能の対象は、現在の `searchResult.items` に含まれる画像のみとする。

```text
対象:
現在画面上に表示されている画像

対象外:
検索条件に一致する全件
DB に登録されている全件
別ページに存在する画像
現在表示されていない画像
```

既存仕様でも、検索・ページ変更・ページサイズ変更・ソート変更時には選択状態をクリアし、見えていない選択を保持しない方針である。
そのため、本機能でもこの設計に合わせる。

## 3.2 バックエンド変更は行わない

本機能はフロントエンド上の選択状態を更新するだけである。

```text
- DB 更新なし
- API 追加なし
- バックエンド DTO 追加なし
- Repository 変更なし
```

選択状態は既存通り `App.vue` の一時状態として扱う。

## 3.3 tile.vue は原則変更しない

既存の `tile.vue` は、チェックボックス表示と `selection-change` emit のみを担当し、選択状態そのものは保持しない設計である。

そのため、本機能では `tile.vue` の変更は原則不要とする。

ただし、チェックボックスの `aria-label` が削除用途に寄っている場合は、次のように汎用化してよい。

```text
画像を選択
```

以下の場合、ボタンは disabled とする。

```text
- isBusy === true
- visibleCount === 0
```

## 5.3 表示位置

推奨表示順は以下とする。

```text

```

既存仕様では、`AppHeader.vue` にワイルドカード出力、ファイル移動、選択画像削除、プロンプト読み取りなどの一括操作が集約されている。
本機能はそれらの前段となる選択操作であるため、一括操作群の左側に配置する。

## 6. App.vue 仕様

## 6.1 現在表示中 ID の算出

`searchResult.items` から現在表示中の画像 ID を算出する。

```js
const visibleImageIds = computed(() =>
  (imageSearch.searchResult.items ?? []).map((item) => item.id)
)
```

実際の `imageSearch.searchResult.items` の参照名は、現行実装に合わせる。

## 6.2 現在表示中件数

```js
const visibleCount = computed(() => visibleImageIds.value.length)
```

## 6.3 表示中画像がすべて選択済みかの判定

```js
const isAllVisibleSelected = computed(() => {
  const ids = visibleImageIds.value

  if (ids.length === 0) {
    return false
  }

  return ids.every((id) => selectedImageIdSet.value.has(id))
})
```

## 6.4 一括選択・解除処理

`isAllVisibleSelected` が `true` の場合は、現在表示中の画像 ID を選択状態から削除する。

`false` の場合は、現在表示中の画像 ID を選択状態へ追加する。

```js
function handleToggleVisibleSelection() {
  const ids = visibleImageIds.value

  if (ids.length === 0) {
    return
  }

  const next = new Set(selectedImageIdSet.value)

  if (isAllVisibleSelected.value) {
    for (const id of ids) {
      next.delete(id)
    }
  } else {
    for (const id of ids) {
      next.add(id)
    }
  }

  selectedImageIdSet.value = next
}
```

## 6.5 既存選択状態との関係

既存の単体チェック操作は維持する。

```text
- 個別チェックボックス操作
- 表示中をすべて選択
- 表示中の選択を解除
```

上記はいずれも同じ `selectedImageIdSet` を更新する。

## 6.6 選択クリア既存仕様は維持

以下のタイミングで選択状態をクリアする既存仕様は変更しない。

```text
- 検索
- ページ変更
- ページサイズ変更
- ソート変更
- 一括削除完了
- 一括移動完了
- ワイルドカード出力成功
```

既存の一括削除仕様でも、再検索・ページ変更・ソート変更などで選択状態をクリアする方針が定義されている。

## 15. 注意事項

## 15.1 「すべて選択」という文言は使わない

ボタン文言は必ず対象範囲を含める。

避ける。

```text
すべて選択
全選択
```

採用する。

```text
表示中をすべて選択
表示中の選択を解除
```

理由は、検索結果全件や DB 全件を対象にしているように誤解されるため。

## 15.2 検索結果全件選択は今回の対象外

今回の仕様には、検索条件に一致する全件の選択は含めない。

検索結果全件選択を行う場合は、現在ページ外の ID 取得 API や、選択対象の永続的な保持方針を別途設計する必要がある。

そこまで広げると、今回のボタン追加ではなく「選択モデルの再設計」になる。チェックボックスひとつが急に町内会長になります。

## 15.3 処理中は選択変更させない

`isBusy` 中は一括選択ボタンを disabled にする。

既存の削除・移動・プロンプト読み取りなどの処理中に選択状態を変更できると、ユーザーが「どの画像を処理したのか」を誤認する可能性がある。

## 15.4 バックエンドへ選択状態を送らない

本機能の時点では、選択状態をバックエンドに送信しない。

バックエンドへ送るのは、既存の一括削除・移動・ワイルドカード出力など、実際に処理を実行するタイミングのみとする。

# 実装仕様書

## 選択画像の一括属性編集機能追加

## 1. 目的

画像一覧画面で選択した複数画像に対して、以下の属性を一括更新できるようにする。

```text
- レーティング rating
- チェック is_checked
- お気に入り is_favorite
```

本機能は、既存の画像選択状態を利用する一括操作であり、既存の一括削除・一括移動・ワイルドカード出力と同じ操作体系に追加する。

タグとコメントは本機能の対象外とする。

```text
対象外:
- tags
- comment
- modelName
- path
- folder
- filename
```

## 2. 前提

既存仕様では、画像選択状態は DB に保存せず、`App.vue` 側の一時状態として保持する構成である。今回もこの方針を維持する。

また、現行構成では `AppHeader.vue`, `MainLayout.vue`, `App.vue`, `useImageMutations.js`, `backendApi.js`, `ImageCatalogApi`, `ImageFileRepository` が一括操作系の主要な影響範囲となっている。

## 3. 変更概要

以下の機能を追加する。

```text
- AppHeader.vue に「選択画像を一括編集」ボタンを追加
- 選択画像 0 件ではボタンを disabled
- ボタン押下で一括属性編集モーダルを表示
- モーダル内で rating / is_checked / is_favorite の更新内容を指定
- 各項目ごとに「変更する / 変更しない」を選択可能にする
- 保存時、有効化された項目だけを選択画像へ一括適用する
- 一括更新は DB トランザクション内で全件成功 / 全件失敗とする
- 保存成功後、選択状態をクリアする
- 保存成功後、現在検索条件で再検索する
```

## 5. 基本方針

## 5.1 選択状態は既存仕様を流用する

本機能用の DB カラムは追加しない。

```text
image_file_data に一括編集用の選択状態を持たせない
```

既存の `selectedImageIds` / `selectedImageIdSet` をそのまま利用する。

## 5.2 更新対象項目は明示的に選択する

一括編集モーダルでは、各項目に対して次の状態を持たせる。

```text
- 変更しない
- 変更する
```

`変更する` が有効な項目のみ、保存時の `updates` に含める。

理由は、たとえばレーティングだけ変更したい場合に、チェック・お気に入りまで意図せず上書きされる事故を避けるためである。

## 5.3 updates に含まれない項目は更新しない

API payload の `updates` に存在しない項目は、DB 更新対象外とする。

例: レーティングのみ更新。

```json
{
  "ids": [1, 2, 3],
  "updates": {
    "rating": "R-18"
  }
}
```

この場合、`is_checked` と `is_favorite` は変更しない。

## 5.4 一括更新は全件成功 / 全件失敗

本機能は DB 更新のみで完結するため、一括更新全体を 1 トランザクションで扱う。

```text
成功:
- 対象画像すべてへ更新を適用する

失敗:
- rollback し、対象画像を一切更新しない
```

物理ファイル削除やファイル移動とは異なり、ファイルシステム操作を伴わないため、部分成功は扱わない。

## 5.5 保存成功後は選択状態をクリアする

既存の一括操作と同様、保存成功後は選択状態を解除する。

```text
- モーダルを閉じる
- 選択状態をクリアする
- 現在検索条件で再検索する
```

## 7.8 saveBulkAttributeEdit

入力。

```js
saveBulkAttributeEdit({
  ids,
  refresh,
  clearSelection,
})
```

処理フロー。

```text
1. ids が空なら warning toast
2. updates を生成する
3. updates が空なら warning toast
4. isSaving = true
5. bulkUpdateImageFileAttributes API を呼ぶ
6. 成功時 toast 表示
7. clearSelection() を実行
8. refresh() で現在条件のまま再検索
9. モーダルを閉じる
10. finally で isSaving = false
```

実装例。

```js
import { bulkUpdateImageFileAttributes } from '../services/backendApi'

async function saveBulkAttributeEdit({ ids, refresh, clearSelection }) {
  const targetIds = [...(ids ?? [])]

  if (targetIds.length === 0) {
    pushToast({
      type: 'warning',
      message: '一括編集する画像を選択してください。',
    })
    return
  }

  const updates = buildBulkAttributeUpdates(bulkAttributeEditModal.form)

  if (Object.keys(updates).length === 0) {
    pushToast({
      type: 'warning',
      message: '変更する項目を選択してください。',
    })
    return
  }

  bulkAttributeEditModal.isSaving = true

  try {
    const result = await bulkUpdateImageFileAttributes({
      ids: targetIds,
      updates,
    })

    if (!result?.success) {
      pushToast({
        type: 'danger',
        message: result?.message || '選択画像の属性更新に失敗しました。',
      })
      return
    }

    const data = result.data ?? {}

    pushToast({
      type: 'success',
      message: `選択画像の属性を更新しました。対象: ${data.targetCount ?? targetIds.length} 件 / 更新: ${data.updatedCount ?? 0} 件`,
    })

    clearSelection?.()
    await refresh?.()
    closeBulkAttributeEditModal()
  } catch (error) {
    pushToast({
      type: 'danger',
      message: '選択画像の属性更新に失敗しました。',
    })
  } finally {
    bulkAttributeEditModal.isSaving = false
  }
}
```

## 14.2 ID 正規化

既存の `_normalize_record_ids(value)` または `_normalize_delete_ids(value)` と同等の正規化を利用する。

```text
- ids が list でない場合は空配列
- int 変換できない値は除外
- 0 以下は除外
- 重複 ID は除外
```

既存の `ImageCatalogApi` には一括削除・一括移動向けの ID 正規化メソッドが存在するため、可能であればそれを流用する。

## 14.3 updates 正規化

```python
def _normalize_bulk_attribute_updates(self, value) -> dict:
    if not isinstance(value, dict):
        return {}

    updates: dict = {}

    if "rating" in value:
        rating = str(value.get("rating") or "").strip()

        if rating not in ("General", "R-15", "R-18", "R-18G"):
            raise ValueError("不正なレーティングが指定されています。")

        updates["rating"] = rating

    if "isChecked" in value:
        updates["is_checked"] = self._normalize_flag_value(value.get("isChecked"))

    if "isFavorite" in value:
        updates["is_favorite"] = self._normalize_flag_value(value.get("isFavorite"))

    return updates
```

## 14.4 フラグ値正規化

```python
def _normalize_flag_value(self, value) -> int:
    try:
        normalized = int(value)
    except (TypeError, ValueError):
        raise ValueError("不正なフラグ値が指定されています。")

    if normalized not in (0, 1):
        raise ValueError("不正なフラグ値が指定されています。")

    return normalized
```

## 14.5 bulk_update_image_file_attributes

実装イメージ。

```python
def bulk_update_image_file_attributes(self, payload):
    try:
        data = payload or {}

        record_ids = self._normalize_record_ids(data.get("ids"))

        if not record_ids:
            return ApiResponse(
                success=False,
                message="更新対象が指定されていません。",
                data=self._empty_bulk_attribute_update_data(),
            ).to_dict()

        try:
            updates = self._normalize_bulk_attribute_updates(data.get("updates"))
        except ValueError as error:
            return ApiResponse(
                success=False,
                message=str(error),
                data=self._empty_bulk_attribute_update_data(),
            ).to_dict()

        if not updates:
            return ApiResponse(
                success=False,
                message="更新項目が指定されていません。",
                data=self._empty_bulk_attribute_update_data(),
            ).to_dict()

        repository = self._get_repository()

        updated_count = repository.bulk_update_attributes(
            record_ids=record_ids,
            updates=updates,
        )

        result = BulkAttributeUpdateResult(
            target_count=len(record_ids),
            updated_count=updated_count,
        )

        return ApiResponse(
            success=True,
            message="選択画像の属性を更新しました。",
            data=result.to_api_data(),
        ).to_dict()

    except Exception:
        return ApiResponse(
            success=False,
            message="選択画像の属性更新に失敗しました。",
            data=self._empty_bulk_attribute_update_data(),
        ).to_dict()
```

これ以外のカラムは更新しない。

## 15.4 SQL 方針

`updates` に含まれる項目だけ `SET` 句へ含める。

例: rating と is_checked を更新する場合。

```sql
UPDATE image_file_data
SET
    rating = :rating,
    is_checked = :is_checked
WHERE id IN (:id_0, :id_1, :id_2)
```

## 15.5 実装例

```python
def bulk_update_attributes(
    self,
    record_ids: list[int],
    updates: dict,
) -> int:
    if not record_ids:
        return 0

    allowed_columns = {
        "rating",
        "is_checked",
        "is_favorite",
    }

    update_values = {
        key: value
        for key, value in (updates or {}).items()
        if key in allowed_columns
    }

    if not update_values:
        return 0

    set_clauses = []
    params = {}

    for key, value in update_values.items():
        set_clauses.append(f"{key} = :{key}")
        params[key] = value

    id_placeholders = []

    for index, record_id in enumerate(record_ids):
        param_name = f"id_{index}"
        id_placeholders.append(f":{param_name}")
        params[param_name] = record_id

    sql = f"""
        UPDATE image_file_data
        SET {", ".join(set_clauses)}
        WHERE id IN ({", ".join(id_placeholders)})
    """

    try:
        cursor = self._connection.execute(sql, params)
        updated_count = cursor.rowcount
        self._connection.commit()
        return updated_count
    except Exception:
        self._connection.rollback()
        raise
```

## 15.6 存在しない ID の扱い

指定 ID のうち、DB に存在しない ID は更新されない。

```text
targetCount:
正規化後 ID 件数

updatedCount:
実際に UPDATE された件数
```

通常は画面表示中の選択 ID から送信されるため、差異はほぼ発生しない。
ただし、別操作で削除済みの場合などを考慮し、`updatedCount < targetCount` は許容する。

この場合も API 自体は `success=true` とする。

## 22. 注意点

## 22.1 「チェック」と「選択チェック」を混同しない

本機能には次の2種類のチェックが登場する。

```text
選択チェック:
画像一覧上で一括操作対象にするための UI 状態

is_checked:
画像属性として DB に保存されるチェック状態
```

UI 文言では可能な限り次のように区別する。

```text
選択画像
チェック状態
```

「チェックした画像を一括編集」という文言は、`is_checked` と紛らわしいため避ける。

## 22.2 updates に含めない項目を更新しない

フロントエンド側の初期値をそのまま送信すると、意図しない上書きが発生する。

必ず `変更する` が有効な項目だけ payload に含める。

## 22.3 一括編集はタグ・コメントを対象にしない

タグは種類が増えすぎるため、一括編集対象外とする。

コメントも一律設定のユースケースが弱く、既存コメントの上書き事故が起きやすいため対象外とする。

## 22.4 部分成功は扱わない

本機能は DB 更新のみであり、ファイル操作を伴わない。

そのため、削除・移動のような部分成功集計は不要とする。

```text
更新成功:
success=true

更新失敗:
rollback して success=false
```

ここで部分成功仕様を入れると、機能の割に運用説明が重くなる。小型犬に鞍を乗せる感じです。
