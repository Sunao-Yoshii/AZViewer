# 実装仕様書
## tile.vue 表示・編集フォーム分離対応（200px版）

---

## 1. 目的

画像タイル内の情報表示と編集機能を分離し、
タイル単位で軽量な編集操作を可能にする。

---

## 2. 対象範囲

### フロントエンド

- frontend/src/components/tile/tile.vue
- frontend/src/components/tile/display.vue（新規）
- frontend/src/components/tile/edit.vue（新規）
- frontend/src/styles/app.css

### バックエンド

- backend/services/thumbnail_cache_service.py

---

## 3. コンポーネント構成

```

tile.vue
├─ display.vue
└─ edit.vue

````

---

## 4. tile.vue 仕様

### 4.1 責務

- サムネイル表示
- filename / folder 表示
- 削除ボタン表示
- display / edit の状態切替
- 詳細表示イベント送出
- 削除要求イベント送出
- 保存要求イベント送出

### 4.2 非責務

- API 呼び出し
- DB 更新
- トースト制御
- 再検索制御

---

### 4.3 状態管理

```js
const isEditing = ref(false)
````

---

### 4.4 イベント

| イベント名          | 内容         |
| -------------- | ---------- |
| open-detail    | 詳細モーダル表示   |
| request-delete | 削除要求       |
| save-detail    | 更新要求（親へ委譲） |

---

### 4.5 状態遷移

```
display → Edit押下 → edit
edit → キャンセル → display
edit → 保存成功 → display
edit → 保存失敗 → edit維持
```

## 6. edit.vue 仕様

### 6.1 編集項目

* rating（radio）
* is_checked（トグル）
* is_favorite（トグル）

※ comment は編集対象外

---

### 6.2 初期値

props.item の値をコピーして内部状態として保持する

---

### 6.3 変更判定

対象:

* rating
* is_checked
* is_favorite

```js
isDirty = (
  rating !== item.rating ||
  is_checked !== item.is_checked ||
  is_favorite !== item.is_favorite
)
```

---

### 6.4 戻るボタン仕様

| 状態   | 表示文言   |
| ---- | ------ |
| 未変更  | 表示に戻る  |
| 変更あり | 保存して戻る |

---

### 6.5 ボタン挙動

#### 未変更

* `cancel` emit
* display に戻る

#### 変更あり

* `save` emit

---

### 6.6 emits

```js
emit('cancel')
emit('save', payload)
```

---

### 6.7 保存 payload

```json
{
  "id": 1,
  "rating": "General",
  "is_checked": 1,
  "is_favorite": 0,
  "comment": "既存コメント"
}
```

---

## 7. 保存処理フロー

```
edit.vue
  ↓ emit(save)
tile.vue
  ↓ emit(save-detail)
App.vue
  ↓ update_image_file_detail API
  ↓ 成功 → 再検索
  ↓ 成功 → tile を display に戻す
  ↓ 失敗 → トースト表示
             edit 状態維持
```

---

## 8. エラー時挙動

### 8.1 API失敗時

* edit.vue の状態を維持する
* 入力内容は保持する
* display に戻さない
* 再検索は行わない
* トースト通知を表示する

---

### 8.2 理由

* ユーザー入力の消失防止
* 再編集を可能にするため

## 11. 受け入れ条件

### UI

* タイル幅が 200px で表示される
* display/edit の切替が正しく動作する
* Edit ボタンが表示される

---

### 編集

* 未変更時に保存処理が発生しない
* 変更時のみ保存処理が発生する
* 保存成功で display に戻る
* 保存失敗で edit を維持する

---

### データ

* update_image_file_detail が正しく呼ばれる
* comment が消えない（payloadに含まれる）

---

### UX

* 編集内容が消えない
* 誤操作時に戻れる
* 操作が直感的に理解できる

## 補足（ここは仕様外だけど重要）

今回の設計はちゃんと「tileを賢くしすぎない」に成功してます。

いい感じに「育たないように育ててる」設計です。
