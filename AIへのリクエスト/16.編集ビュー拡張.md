# 実装仕様書

## タイル幅300px化・簡易編集フォーム拡張・詳細モーダル表示専用化

## 1. 目的

画像一覧画面のタイル表示サイズを拡張し、タイル内簡易編集フォームでコメントまで編集できるようにする。
また、`ImageDetailModal.vue` は編集機能を廃止し、画像表示専用ダイアログとして再定義する。

---

### フロントエンド

* `frontend/src/components/tile/tile.vue`
* `frontend/src/components/tile/display.vue`
* `frontend/src/components/tile/edit.vue`

* `frontend/src/components/layout/Content.vue`
* `frontend/src/App.vue`

### バックエンド

* `backend/services/thumbnail_cache_service.py`

* `backend/repositories/image_file_repository.py`

## 3. 変更概要

* `tile.vue` の幅を 300px 固定へ変更する
* サムネイル生成サイズを長辺 300px へ変更する
* 既存サムネイルキャッシュの自動再生成は行わない
* `edit.vue` に `comment` 編集項目を追加する
* `edit.vue` の `rating` を Bootstrap 5.3 の radio button group に変更する
* `ImageDetailModal.vue` を画像表示専用モーダルへ変更する
* `ImageDetailModal.vue` からフォーム類、Save ボタン、フッター、右側領域を削除する
* `ImageDetailModal.vue` のタイトルを `filename` から `path` へ変更する

---

## 4. tile.vue 仕様変更

### 4.1 タイル幅

`tile.vue` のカード幅を 300px 固定とする。

```css
.card.image-tile,
.image-tile {
  width: 300px;
}
```

### 4.2 レイアウト

* 高さはコンテンツ内容に応じて可変とする
* カラム数指定は行わない
* 既存通り、横方向に敷き詰め、横スクロールが発生しない範囲で折り返す
* タイル間余白は既存仕様を維持する
* 既存の一覧表示レイアウトを変更しない

---

## 5. サムネイル仕様変更

### 5.1 生成サイズ

`ThumbnailCacheService.create_thumbnail()` の生成サイズを長辺 300px に変更する。

```python
image.thumbnail((300, 300))
```

### 5.2 生成ルール

* 縦横比を維持する
* 拡大は行わない
* 短辺方向は既存通り中央寄せ表示とする
* 生成形式は既存通り PNG
* 保存先は既存通り `data/thumbs/{id}.png`

### 5.3 既存キャッシュ

* 既存サムネイルキャッシュの自動再生成処理は実装しない
* 既存キャッシュは手動削除前提とする
* 正式リリース前のため、移行処理は不要とする

---

## 6. edit.vue 仕様変更

### 6.1 編集対象

`edit.vue` の編集対象を以下に変更する。

* `rating`
* `is_checked`
* `is_favorite`
* `comment`

既存仕様では `comment` は編集対象外だったが、本仕様で編集対象に追加する。

---

### 6.2 comment 入力

`comment` は textarea として表示する。

#### 入力仕様

* 最大長: 255 文字
* 改行許可
* 空文字許可
* 空文字保存時は DB 上 `NULL` として保存する
* 保存 payload には編集後の `comment` を含める

#### HTML 例

```html
<textarea
  v-model="form.comment"
  class="form-control form-control-sm"
  maxlength="255"
  rows="3"
></textarea>
```

---

### 6.3 rating 入力

`rating` は Bootstrap 5.3 の radio button group を使用する。

#### 選択肢

* `General`
* `R-15`
* `R-18`
* `R-18G`

#### 制約

* 未選択は禁止
* 選択解除は禁止
* 常に 1 つの値が選択されていること
* DB の `NOT NULL` / `CHECK` 制約に違反しないこと

#### 実装方針

* `input type="radio"` と `label.btn` の組み合わせで構成する
* `v-model` は `form.rating` に紐づける
* radio input の `id` は rating 値ベースでよい
* Bootstrap の対象仕様は「チェックボックスとラジオボタングループ」とする

---

### 6.4 内部状態

`props.item` の値をコピーして内部フォーム状態を作成する。

```js
const form = reactive({
  rating: props.item.rating,
  is_checked: props.item.is_checked,
  is_favorite: props.item.is_favorite,
  comment: props.item.comment ?? "",
})
```

---

### 6.5 変更判定

`isDirty` の判定対象に `comment` を追加する。

```js
const isDirty = computed(() => (
  form.rating !== props.item.rating ||
  form.is_checked !== props.item.is_checked ||
  form.is_favorite !== props.item.is_favorite ||
  form.comment !== (props.item.comment ?? "")
))
```

---

### 6.6 戻るボタン仕様

| 状態   | 表示文言   | 動作            |
| ---- | ------ | ------------- |
| 未変更  | 表示に戻る  | `cancel` emit |
| 変更あり | 保存して戻る | `save` emit   |

---

### 6.7 保存 payload

```json
{
  "id": 1,
  "rating": "General",
  "is_checked": 1,
  "is_favorite": 0,
  "comment": "memo"
}
```

## 7. 保存処理フロー

```text
edit.vue
  ↓ save emit
tile.vue
  ↓ save-detail emit
Content.vue
  ↓ save-detail emit
App.vue
  ↓ update_image_file_detail API
  ↓ 成功時: 再検索
  ↓ 成功時: edit状態解除
  ↓ 失敗時: トースト表示、edit状態維持
```

### 7.1 成功時

* `update_image_file_detail` を呼び出す
* 成功時は再検索する
* 再検索によりタイル状態は通常表示へ戻る

### 7.2 失敗時

* edit 表示を維持する
* 入力内容を保持する
* トースト通知を表示する
* 再検索は行わない

## 8. ImageDetailModal.vue 再定義

### 8.1 役割

`ImageDetailModal.vue` は画像表示専用モーダルとする。
編集機能は持たない。

---

### 8.2 削除する機能

以下を削除する。

* `rating` 編集フォーム
* `is_checked` 編集フォーム
* `is_favorite` 編集フォーム
* `comment` 編集フォーム
* Save ボタン
* 保存確認処理
* `save` emit
* `isSaving` prop
* モーダル右側フォーム領域
* モーダルフッター

---

### 8.3 タイトル

モーダルタイトルは `filename` ではなく `path` を表示する。

```html
<h5 class="modal-title detail-modal-title">
  {{ item.path }}
</h5>
```

長いパスでレイアウトが崩れないようにする。

```css
.detail-modal-title {
  overflow-wrap: anywhere;
  word-break: break-word;
}
```

---

### 8.4 本文レイアウト

* モーダル本文は画像表示領域のみとする
* 右側領域は設けない
* フォーム領域は設けない
* 画像は中央表示する
* 必要に応じて縦スクロールを許可する
* ダイアログの縦方向最大表示サイズは表示ウィンドウの 90% までとする

---

### 8.5 画像表示

* `fetchLocalImage(path)` で画像本体を取得する
* 画像クリックによる縮小表示 / 原寸表示の切替は既存仕様を維持する
* 画像表示領域はモーダル本文全体を使用する

---

## 9. App.vue 側の変更

### 9.1 維持する処理

`handleSaveDetail(payload)` は削除しない。
`edit.vue` からの保存要求で引き続き使用する。

### 9.2 削除・変更する接続

* `ImageDetailModal.vue` への `isSaving` prop 渡しを削除する
* `ImageDetailModal.vue` からの `save` イベント受信を削除する
* 詳細モーダルでは `open` / `close` / 画像読み込みのみ扱う

---

## 10. バックエンド仕様

### 10.1 update_image_file_detail

既存の詳細更新 API を継続利用する。

#### 入力

```json
{
  "id": 1,
  "rating": "General",
  "is_checked": 1,
  "is_favorite": 0,
  "comment": "memo"
}
```

### 10.2 comment 正規化

* `comment` が空文字の場合、DB 保存値は `NULL` とする
* `comment` が文字列の場合、最大 255 文字以内で保存する
* 改行は許可する

---

## 11. 受け入れ条件

### 11.1 タイル

* tile 幅が 300px で表示されること
* 既存同様、横スクロールせず折り返し表示されること
* タイル間余白が既存から変わらないこと
* タイル高さは内容に応じて可変であること

### 11.2 サムネイル

* 新規生成サムネイルが長辺 300px になること
* 縦横比が維持されること
* 小さい画像が拡大されないこと
* 既存キャッシュ自動再生成処理が追加されていないこと

### 11.3 edit.vue

* `comment` textarea が表示されること
* `comment` が最大 255 文字であること
* `comment` の改行が入力できること
* `comment` 空文字保存時に DB 上 `NULL` として保存されること
* `comment` 変更時も「保存して戻る」扱いになること
* `rating` が Bootstrap radio button group で表示されること
* `rating` を未選択にできないこと
* 保存失敗時に edit 表示が維持されること

### 11.4 ImageDetailModal.vue

* フォーム類が表示されないこと
* Save ボタンが表示されないこと
* フッターが表示されないこと
* 右側領域が存在しないこと
* 本文領域が画像表示のみであること
* タイトルに `path` が表示されること
* 長い `path` でもレイアウトが崩れないこと
* 画像表示領域がモーダル内で適切にスクロール可能であること

---

## 12. 注意事項

* `ImageDetailModal.vue` は編集機能を完全に持たない
* 編集は `edit.vue` に集約する
* `handleSaveDetail` はモーダル用ではなく、タイル内編集用として維持する
* 既存サムネイルキャッシュの更新は手動削除で対応する
* tile レイアウトの折り返し挙動、余白、横スクロール抑止は変更しない
