# 実装仕様書

## Stable Diffusion WebUI プロンプト一括タグ登録機能

## 1. 目的

画像ファイルのメタデータから Stable Diffusion WebUI のプロンプト情報を読み取り、タグ未登録の画像に対して一括でタグを登録する。

本機能は、既存のタグ管理機能を補助する一括処理であり、手動でタグ登録済みの画像には影響を与えない。

---

## 2. 基本方針

### 2.1 対象画像

処理対象は以下を満たす `image_file_data` のみとする。

```text
tag_image_link が 1 件も存在しない image_file_data レコード
```

既に 1 件以上のタグが紐づいている画像は、手動で意図したタグが存在する可能性があるため完全に対象外とする。

---

### 2.2 メタ情報読み取り

画像メタ情報の読み取りは、既存の `ImageMetadataService` を利用する。

現在の `ImageMetadataService` は、Pillow で画像ファイルを開き、PNG の `image.info` や JPEG/JPG/WebP/AVIF の EXIF を YAML 風プレーンテキストへ整形し、EXIF `UserComment` についても接頭辞や候補エンコーディングを解釈して Unicode 化する役割を持つ。

そのため、本機能側では EXIF bytes の文字コード判定を再実装しない。

---

### 2.3 タグ正規化

抽出したプロンプト文字列は、既存の `TagNormalizeService` に渡してタグ化する。

既存の `TagNormalizeService` は、カンマ分割、非エスケープ括弧除去、重み表記除去、小文字化、重複除去、長さ検証を担当している。

## 4.1 Repository 追加

`ImageFileRepository` に、タグ未登録画像を取得するメソッドを追加する。

```python
def find_items_without_tags(self) -> list[ImageFileListItem]:
    ...
```

### SQL 方針

```sql
SELECT i.*
FROM image_file_data i
WHERE NOT EXISTS (
    SELECT 1
    FROM tag_image_link l
    WHERE l.image_file_id = i.id
)
ORDER BY i.id ASC;
```

### 注意

`tag_image_link` が 1 件でも存在する画像は対象外とする。

## 4.2 ImageMetadataService 追加

`ImageMetadataService` に、Stable Diffusion WebUI のプロンプト抽出用公開メソッドを追加する。

```python
def extract_stable_diffusion_prompt(self, path: str) -> str | None:
    ...
```

### 役割

```text
1. 画像ファイルを開く
2. 既存の _build_metadata_list を利用してメタ情報行リストを取得する
3. 拡張子に応じて対象キーを探す
4. 対象キーの値を復元する
5. Positive prompt 部分のみ抽出して返す
```

### 対象キー

| 拡張子     | 対象キー               |
| ------- | ------------------ |
| `.png`  | `parameters`       |
| `.jpg`  | `exif.UserComment` |
| `.jpeg` | `exif.UserComment` |
| `.webp` | `exif.UserComment` |
| `.avif` | `exif.UserComment` |

---

## 4.3 YAML 風ブロックの復元

`ImageMetadataService` のメタ情報表示は YAML 風プレーンテキストであり、複数行値はブロック形式になる可能性がある。

次の両方に対応する。

### 1行形式

```text
parameters: masterpiece, best quality, 1girl, Negative prompt: low quality
```

### ブロック形式

```text
parameters: |
  masterpiece, best quality, 1girl
  Negative prompt: low quality
  Steps: 28, Sampler: DPM++ 2M
```

ブロック形式の場合、対象キー行の次行以降にあるインデント行を本文として連結する。

---

## 4.4 Positive prompt 抽出ルール

抽出した本文から、以下の優先順位で Positive prompt を切り出す。

### 4.4.1 `Negative prompt:` が存在する場合

```text
本文先頭 ～ Negative prompt: の直前
```

### 4.4.2 `Negative prompt:` が存在せず、`Steps:` が存在する場合

```text
本文先頭 ～ Steps: の直前
```

### 4.4.3 `Negative prompt:` も `Steps:` も存在しない場合

```text
本文全体
```

## 4.5 BREAK 除去ルール

タグ正規化前に、プロンプト文字列から `BREAK` を除去する。

### 条件

```text
大文字小文字を区別しない
単語として独立している BREAK のみ除去する
breakdown など、単語の一部は除去しない
```

### 実装例

Python 側:

```python
import re

def remove_break_token(value: str) -> str:
    return re.sub(
        r"(?<![A-Za-z0-9_])break(?![A-Za-z0-9_])",
        "",
        value,
        flags=re.IGNORECASE,
    )
```

## 4.7 処理フロー

```text
1. find_items_without_tags() で対象画像を取得する
2. targetCount を記録する
3. 対象画像を 1 件ずつ処理する
4. ImageMetadataService.extract_stable_diffusion_prompt(path) を呼ぶ
5. プロンプトが取得できなければ skippedCount 加算
6. BREAK を除去する
7. TagNormalizeService.normalize_tags([prompt]) を呼ぶ
8. 正規化後タグが 0 件なら skippedCount 加算
9. repository.replace_tags(image_file_id, tags) でタグ登録する
10. 成功したら taggedCount 加算
11. 例外発生時は failedCount 加算し、失敗詳細を最大 20 件まで保持する
12. 全件処理後、結果 DTO を返す
```

---

## 4.8 トランザクション方針

一括全体を 1 トランザクションにはしない。

```text
1画像ごとにタグ登録を完結させる
1画像で失敗しても全体処理は継続する
```

理由は、画像数が多い場合に 1 件の壊れた画像で全体がロールバックされることを防ぐため。

## 4.9 戻り値 DTO

`backend/models/image_file.py` に以下を追加する。

```python
@dataclass
class PromptTagImportFailure:
    id: int
    path: str
    reason: str

    def to_dict(self) -> dict:
        return {
            "id": self.id,
            "path": self.path,
            "reason": self.reason,
        }


@dataclass
class PromptTagImportResult:
    target_count: int
    processed_count: int
    tagged_count: int
    skipped_count: int
    failed_count: int
    failed_files: list[PromptTagImportFailure]

    def to_api_data(self) -> dict:
        return {
            "targetCount": self.target_count,
            "processedCount": self.processed_count,
            "taggedCount": self.tagged_count,
            "skippedCount": self.skipped_count,
            "failedCount": self.failed_count,
            "failedFiles": [
                item.to_dict()
                for item in self.failed_files[:20]
            ],
        }
```

### 失敗詳細上限

```text
failedFiles は最大 20 件まで返す
```

```javascript
export function usePromptTagImport({

  refresh,

    ...
  }

  return {
    handleImportPromptTags,
  }

```

### 処理フロー

```text
1. confirm を表示する
2. キャンセルなら何もしない
3. LoadingOverlay を表示する
4. importPromptTags() を呼ぶ
5. 成功時は結果件数を toast 表示する

7. 完了後 refresh() で現在条件のまま再検索する
8. finally で LoadingOverlay を解除する
```

### 確認文言

```text
タグ未登録の画像から Stable Diffusion WebUI のプロンプト情報を読み取り、タグとして登録します。
既にタグが登録されている画像は対象外です。
実行しますか？
```

### LoadingOverlay 文言

```text
タイトル: プロンプト情報を読み取り中
メッセージ: 画像メタデータからタグを一括登録しています...
```

既存の `useLoadingOverlay.js` は全画面ローディングオーバーレイの状態管理を担当しており、`showLoading`, `hideLoading`, `withLoading` を公開する構成です。

## 6. エラー処理

### 6.1 画像単位エラー

1画像の処理でエラーが発生した場合、その画像のみ失敗扱いにし、次の画像へ進む。

対象例:

```text
ファイル不存在
画像を開けない
メタ情報読み取り失敗
タグ正規化エラー
DB 登録失敗
```

---

### 6.2 全体エラー

Repository 初期化失敗など、一括処理自体を開始できない場合は `success: false` を返す。

ただし、フロントエンド側では `finally` により必ず LoadingOverlay を解除する。

## 7. 受け入れ条件

### 7.1 UI

```text
AppHeader.vue に「プロンプトの読み取り」ボタンが表示されること
処理中はボタンが disabled になること
押下時に確認ダイアログが表示されること
処理中は LoadingOverlay が表示されること
処理完了時に LoadingOverlay が解除されること
```

### 7.2 対象抽出

```text
tag_image_link が存在しない画像のみ対象になること
tag_image_link が 1 件以上ある画像は対象外になること
```

### 7.3 メタ情報抽出

```text
PNG の parameters からプロンプトを抽出できること
JPG/JPEG/WebP/AVIF の exif.UserComment からプロンプトを抽出できること
1行形式と YAML 風ブロック形式の両方に対応すること
```

### 7.4 Positive prompt 判定

```text
Negative prompt: がある場合、その直前までをタグ対象にすること
Negative prompt: がなく Steps: がある場合、Steps: 直前までをタグ対象にすること
Negative prompt: も Steps: もない場合、全文をタグ対象にすること
```

### 7.5 BREAK 除去

```text
BREAK / break / Break など大文字小文字違いを除去できること
breakdown など単語の一部は除去しないこと
```

### 7.6 タグ登録

```text
既存の TagNormalizeService を通してタグ登録されること
カンマ分割、重み除去、小文字化、重複除去、128文字制限が既存仕様と一致すること
タグが 1 件以上抽出できた画像のみ tag_image_link が作成されること
```

### 7.7 結果通知

```text
対象件数、処理件数、タグ登録件数、スキップ件数、失敗件数が返ること
失敗詳細は最大20件まで返ること
完了後に現在条件のまま画像一覧が再検索されること
```

## 8. 注意点

`ImageMetadataService._build_metadata_list` は既存の内部メソッドなので、今回の一括処理から直接呼ぶのではなく、`extract_stable_diffusion_prompt(path)` のような公開メソッドを追加して、その内部で利用する形がよいです。内部メソッド直呼びは、後で「private とは？」という哲学カフェが始まります。
