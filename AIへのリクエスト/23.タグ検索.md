# 実装設計書

## タグAND検索機能追加

## 1. 目的

画像検索フォームにタグ検索条件を追加し、登録済み画像をタグで絞り込めるようにする。

タグは最大3件まで指定可能とし、複数指定時は AND 条件として扱う。

```text
例:
tags = ["blue eyes", "smile"]

→ blue eyes と smile の両方を持つ画像のみ検索対象
```

本機能は、既存の `tag` / `tag_image_link` によるタグ管理機能を検索条件として利用するものであり、タグ登録・タグ編集仕様は変更しない。

---

## 2. 前提

既存のタグ管理機能では、タグは `tag` マスタテーブルと `tag_image_link` リンクテーブルで管理される。`tag.name` は一意で、`tag_image_link` は `image_file_id` と `tag_id` の複合主キーを持つ構成である。

既存の検索関連構成は以下を前提とする。

```text
frontend/src/components/form/search_form.vue
frontend/src/composables/useImageSearch.js
frontend/src/services/backendApi.js
backend/api/app_api.py
backend/api/image_catalog_api.py
backend/repositories/image_file_repository.py
```

`AppApi` は pywebview から Vue に公開する統合 API であり、画像検索などを各 API へ委譲する構成である。
`ImageCatalogApi` は画像一覧検索、詳細更新、削除、画像取得、メタ情報取得などを担当する API である。
`ImageFileRepository` は `image_file_data`, `tag`, `tag_image_link` に対する CRUD、検索、タグ関連付けを担当している。

---

### 5.2.6 表示件数

タグ一覧の上部に、現在表示中の件数と、現在の条件に一致するタグ総数を表示する。

表示形式。

```text
表示件数: {表示中タグ数} / {条件一致総数}
```

例。

```text
表示件数: 12 / 42
```

条件一致総数が 256 件を超える場合。

```text
表示件数: 256 / 1840
```

この場合、タグ一覧には最大256件までしか表示されない。

### 5.2.7 タグ一覧

タグ一覧は `tag` テーブルから取得する。

条件。

```text
- ID 昇順
- 最大256件
- フィルタ指定時は name 部分一致
```

タグは badge 形式で表示する。

```html
<span class="badge text-bg-light border">
  blue eyes
</span>
```

クリックすると、選択中タグへ追加する。

### 5.2.8 最大選択数

選択可能なタグは最大3件。

3件選択済みの状態で未選択タグをクリックした場合は、追加しない。

このとき、ダイアログ内にインラインメッセージまたはトーストを表示してよい。

```text
タグは最大3件まで指定できます。
```

### 5.2.9 重複選択

既に選択中のタグをクリックしても追加しない。

挙動は以下のどちらでもよい。

```text
- 何もしない
- 選択済み扱いの見た目にする
```

実装としては「何もしない」でよい。

### 5.2.10 OKボタン

OKボタン押下時、ダイアログ内の仮選択タグを `search_form.vue` の `filters.tags` に反映し、ダイアログを閉じる。

OK押下だけでは画像検索は実行しない。
画像検索は既存の検索ボタン押下で実行する。

### 5.2.11 キャンセル

キャンセルまたは右上 × で閉じた場合、`search_form.vue` の `filters.tags` は変更しない。

## 8.2 ImageCatalogApi

### 8.2.1 search_image_files 変更

payload から `tags` を取得し、検索条件として repository へ渡す。

```python
def search_image_files(self, payload):
    data = payload or {}

    tags = self._normalize_search_tags(data.get("tags"))

    result = repository.search(
        path=path,
        rating=rating,
        is_checked=is_checked,
        is_favorite=is_favorite,
        tags=tags,
        page=page,
        page_size=page_size,
        sort=sort,
    )
```

### 8.2.2 _normalize_search_tags 追加

```python
def _normalize_search_tags(self, value) -> list[str]:
    if not isinstance(value, list):
        return []

    tags: list[str] = []

    for item in value:
        tag = str(item or "").strip()
        if not tag:
            continue
        if tag in tags:
            continue
        tags.append(tag)

    return tags[:3]
```

検索用タグはタグ検索ダイアログから選択されるため、保存時のタグ正規化処理は原則不要とする。
ただし、空文字・重複・件数超過は防御的に処理する。

### 8.2.3 fetch_tags_for_search 追加

```python
def fetch_tags_for_search(self, payload):
    try:
        data = payload or {}
        keyword = str(data.get("keyword") or "").strip()

        try:
            limit = int(data.get("limit") or 256)
        except (TypeError, ValueError):
            limit = 256

        limit = max(1, min(limit, 256))

        repository = self._get_repository()

        tags = repository.find_tags_for_search(
            keyword=keyword,
            limit=limit,
        )
        total_count = repository.count_tags_for_search(
            keyword=keyword,
        )

        return ApiResponse(
            success=True,
            message="",
            data={
                "tags": [tag.to_dict() for tag in tags],
                "total_count": total_count,
                "limit": limit,
            },
        ).to_dict()

    except Exception:
        return ApiResponse(
            success=False,
            message="タグ一覧を取得できませんでした。",
            data={
                "tags": [],
                "total_count": 0,
                "limit": 256,
            },
        ).to_dict()
```

### 8.4.2 タグAND条件

タグ条件がある場合のみ、`image_file_data.id` をサブクエリで絞り込む。

```sql
i.id IN (
    SELECT
        til.image_file_id
    FROM tag_image_link til
    INNER JOIN tag t
        ON t.id = til.tag_id
    WHERE t.name IN (:tag_0, :tag_1, :tag_2)
    GROUP BY til.image_file_id
    HAVING COUNT(DISTINCT t.name) = :tag_count
)
```

実際には指定タグ数に応じて placeholder を生成する。

```python
tag_conditions = []

for index, tag in enumerate(tags):
    key = f"tag_{index}"
    params[key] = tag
    tag_conditions.append(f":{key}")

where_clauses.append(f"""
    i.id IN (
        SELECT
            til.image_file_id
        FROM tag_image_link til
        INNER JOIN tag t
            ON t.id = til.tag_id
        WHERE t.name IN ({", ".join(tag_conditions)})
        GROUP BY til.image_file_id
        HAVING COUNT(DISTINCT t.name) = :tag_count
    )
""")

params["tag_count"] = len(tags)
```

### 8.4.3 count SQL への反映

一覧取得 SQL だけでなく、`total_count` 算出 SQL にも同じタグ条件を適用する。

```text

- total_count 取得SQL
```

ここを分けて書くと、検索結果は正しいのにページ数だけ嘘をつく。UIとしてはかなり陰湿な嘘です。

```python
tag_count = 2
```

この場合、`blue eyes` と `smile` の両方を持つ画像のみ一致する。

## 10. 検索条件保持

タグ条件は既存検索条件と同様に保持する。

```text
- 検索時: page = 1
- ページ変更時: tags 条件を維持
- ページサイズ変更時: tags 条件を維持し、page = 1
- ソート変更時: tags 条件を維持
```

既存仕様でも、検索時はページ1へ戻し、ページ移動時は条件を保持する方針である。


## 12.2 TagSearchModal.vue

```text
- タグ選択ボタン押下でタグ検索ダイアログが開くこと

- 検索ボタン押下でタグ候補検索が実行されること
- Enter キー押下でタグ候補検索が実行されること
- タグ一覧上部に「表示件数: 表示中件数 / 条件一致総数」が表示されること
- タグ一覧は ID 昇順で最大256件まで表示されること
- タグ badge クリックで選択中タグへ追加できること
- 4件目のタグは追加できないこと
- OK押下で search_form.vue の filters.tags に反映されること
- キャンセル時は search_form.vue の filters.tags が変更されないこと
```

## 12.3 API

```text
- fetch_tags_for_search が追加されていること
- keyword 空指定で tag を ID 昇順、最大256件取得できること
- keyword 指定で tag.name の部分一致検索ができること
- fetch_tags_for_search が total_count を返すこと
- total_count は limit 適用前の条件一致総数であること
```

## 12.4 Repository

```text
- ImageFileRepository.search が tags 条件を受け取れること
- tags 未指定時は既存検索結果に影響しないこと
- tags 1件指定時、そのタグを持つ画像のみ検索されること
- tags 複数指定時、指定タグをすべて持つ画像のみ検索されること
- total_count にも tags 条件が反映されること
- 検索結果 item の tags 付与が維持されること
- タグ取得が N+1 クエリにならないこと
```

## 14. 注意点

`fetch_tags_for_search` の `total_count` は、`LIMIT 256` 適用後の件数ではなく、現在のフィルタ条件に一致する総数とする。ここを間違えると「表示件数: 256 / 256」になり、実際には 1840 件あるのに全部見えているように見える。これは小さな数字の顔をした大きな嘘です。

タグ検索条件は `tag.name` ベースでよい。既存の一覧 item も `tags: list[str]` として扱う設計であり、表示・編集もタグ名中心の構成であるため。将来的にタグ名編集機能を追加する場合は、`tag_id` ベース検索への変更を検討する。
