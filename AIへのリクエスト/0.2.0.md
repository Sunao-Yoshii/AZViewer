# 実装設計書

## 画像メタ情報表示・選択テキストコピー・タグ流用機能追加

## 1. 目的

画像ファイルに含まれるメタ情報をプレーンテキストとして表示し、ユーザーが任意範囲をテキスト選択できるようにする。

選択したテキストは以下の用途に利用する。

* クリップボードへコピーする
* Stable Diffusion WebUI 等のプロンプト文字列とみなし、画像タグ入力欄へ流用する

本機能は、既存の画像タグ管理機能を補助するための機能である。
タグ保存自体は既存の `edit.vue` の保存フローに従い、メタ情報モーダルから直接 DB 保存は行わない。

## 2. 基本方針

### 2.1 ImageDetailModal.vue とは分離する

既存の `ImageDetailModal.vue` は画像表示専用モーダルとして定義済みであり、フォーム類や保存機能を持たない方針である。

そのため、本機能では `ImageDetailModal.vue` を拡張せず、新規にメタ情報表示専用モーダルを追加する。

### 2.2 タグ保存は既存フローを維持する

既存のタグ管理機能では、`edit.vue` でタグを編集し、`update_image_file_detail` の payload に `tags` を含めて保存する構成である。

本機能の「タグ付け」ボタンは、選択テキストを `edit.vue` のタグ入力欄へ反映するだけとする。
DB 保存は行わない。

### 2.3 API 呼び出しは composable に集約する

既存のフロントエンドリファクタ方針では、コンポーネントから `backendApi.js` を直接呼ばず、API 呼び出しは composable に集約する方針である。

本機能でもこの方針に従い、メタ情報取得処理は `useImageMetadata.js` に集約する。

## 4. 機能概要

### 4.1 display.vue からの利用

`display.vue` にメタ情報表示ボタンを追加する。

```text

```

クリック時、メタ情報モーダルを表示する。

display モードでは、モーダル内に以下を表示する。

* メタ情報 textarea
* コピーボタン
* 閉じるボタン

タグ付けボタンは表示しない。

### 4.2 edit.vue からの利用

`edit.vue` にメタ情報表示ボタンを追加する。

```text
メタ情報からタグ
```

クリック時、メタ情報モーダルを表示する。

edit モードでは、モーダル内に以下を表示する。

* メタ情報 textarea
* コピー ボタン
* タグ付け ボタン
* 閉じる ボタン

### 4.3 タグ付けの動作

`ImageMetadataModal.vue` 上でテキスト選択した状態で「タグ付け」を押下した場合、以下の処理を行う。

```text
1. textarea の選択範囲テキストを取得する
2. 選択テキストから改行コードなどの制御文字を除去する
3. 選択テキストをクリップボードへコピーする
4. モーダルを閉じる
5. edit.vue の tags テキストボックスへ反映する
```

この時点では `form.tags` には追加しない。
ユーザーが tagInput の内容を確認し、既存仕様通り Enter 操作等でタグ追加する。

```js
async function openMetadataModal(item, mode = 'display') {
  metadataModal.show = true
  metadataModal.mode = mode
  metadataModal.item = item
  metadataModal.metadataText = ''
  metadataModal.isLoading = true

  try {
    const result = await fetchImageMetadata(item.path)

    if (!result?.success) {
      metadataModal.metadataText = ''
      pushToast({
        type: 'danger',
        message: result?.message || 'メタ情報を取得できませんでした',
      })
      return
    }

    metadataModal.metadataText = result.data?.metadata ?? ''
  } catch (error) {
    metadataModal.metadataText = ''
    pushToast({
      type: 'danger',
      message: 'メタ情報を取得できませんでした',
    })
  } finally {
    metadataModal.isLoading = false
  }
}
```

### 5.2.7 handleCopyMetadataText

```js
async function handleCopyMetadataText({ text }) {
  const value = text ?? ''

  if (!value) {
    pushToast({
      type: 'warning',
      message: 'テキストが選択されていません',
    })
    return false
  }

  try {
    await navigator.clipboard.writeText(value)

    pushToast({
      type: 'success',
      message: '選択テキストをコピーしました',
    })

    return true
  } catch (error) {
    pushToast({
      type: 'danger',
      message: 'クリップボードへのコピーに失敗しました',
    })

    return false
  }
}
```

### 5.2.8 handleApplyMetadataTags

```js
async function handleApplyMetadataTags({ text }, applyToTagInput) {
  const value = sanitizeMetadataSelection(text)

  if (!value) {
    pushToast({
      type: 'warning',
      message: 'テキストが選択されていません',
    })
    return
  }

  try {
    await navigator.clipboard.writeText(value)
  } catch (error) {
    // タグ入力への反映は継続する
    pushToast({
      type: 'warning',
      message: 'コピーには失敗しましたが、タグ入力欄へ反映します',
    })
  }

  applyToTagInput(value)

  closeMetadataModal()

  pushToast({
    type: 'success',
    message: '選択テキストをタグ入力欄へ反映しました',
  })
}
```

### 5.2.9 制御文字除去

```js
function sanitizeMetadataSelection(value) {
  return String(value ?? '')
    .replace(/[\r\n\t]/g, ' ')
    .replace(/[\x00-\x1F\x7F]/g, '')
    .replace(/\s+/g, ' ')
    .trim()
}
```

### 注意

この処理では、カンマ `,` は除去しない。
既存タグ仕様では ASCII カンマのみがタグ区切り文字であるため、Stable Diffusion WebUI のプロンプト文字列をそのまま tagInput に流用できる。

## 6.1 ImageMetadataService

### 6.1.1 配置

```text
backend/services/image_metadata_service.py
```

### 6.1.2 役割

画像ファイルから取得可能なメタ情報を読み取り、表示用プレーンテキストへ変換する。

### 6.1.3 方針

* Pillow を使用して画像メタ情報を取得する
* 厳密な YAML としての構文保証はしない
* YAML 風のプレーンテキストとして整形する
* 取得できない項目は出力しない
* 画像ファイルが存在しない場合はエラーとする

### 6.1.4 クラス定義案

```python
from __future__ import annotations

from pathlib import Path
from PIL import Image


class ImageMetadataService:
    """画像ファイルのメタ情報を表示用テキストとして取得する。"""

    def fetch_metadata_text(self, path: str) -> str:
        image_path = Path(path)

        if not image_path.exists():
            raise FileNotFoundError(f"画像ファイルが存在しません: {path}")

        if not image_path.is_file():
            raise ValueError(f"画像ファイルではありません: {path}")

        with Image.open(image_path) as image:
            return self._build_metadata_text(image_path, image)

    def _build_metadata_text(self, image_path: Path, image: Image.Image) -> str:
        lines: list[str] = []

        lines.append(f"filename: {image_path.name}")
        lines.append(f"path: {image_path}")
        lines.append(f"format: {image.format or ''}")
        lines.append(f"mode: {image.mode}")
        lines.append(f"width: {image.width}")
        lines.append(f"height: {image.height}")

        info = dict(image.info or {})

        for key, value in info.items():
            lines.extend(self._format_metadata_value(key, value))

        return "\n".join(lines)

    def _format_metadata_value(self, key: str, value) -> list[str]:
        text = self._to_text(value)

        if "\n" in text:
            return [
                f"{key}: |",
                *[f"  {line}" for line in text.splitlines()],
            ]

        return [f"{key}: {text}"]

    def _to_text(self, value) -> str:
        if isinstance(value, bytes):
            try:
                return value.decode("utf-8", errors="replace")
            except Exception:
                return repr(value)

        return str(value)
```

```python
def fetch_image_metadata(self, payload):
    try:
        path = (payload or {}).get("path")

        if not path:
            return ApiResponse(
                success=False,
                message="path が指定されていません。",
                data={"metadata": ""},
            ).to_dict()

        metadata = self._metadata_service.fetch_metadata_text(path)

        return ApiResponse(
            success=True,
            message="",
            data={"metadata": metadata},
        ).to_dict()

    except FileNotFoundError:
        return ApiResponse(
            success=False,
            message="画像ファイルが存在しません。",
            data={"metadata": ""},
        ).to_dict()

    except Exception as error:
        return ApiResponse(
            success=False,
            message="メタ情報を取得できませんでした。",
            data={"metadata": ""},
        ).to_dict()
```

# 8. タグ入力への反映仕様

## 8.1 反映先

`edit.vue` の `form.tagInput` に反映する。

## 8.2 反映時の加工

選択テキストに対して以下を行う。

```text
1. CR / LF / TAB を半角スペースへ変換
2. その他 ASCII 制御文字を除去
3. 連続空白を 1 つへ圧縮
4. 前後空白を trim
```

## 8.3 行わないこと

以下は行わない。

* タグ正規化
* 小文字化
* 禁止文字除去
* `:1.2` 形式の重み除去
* tag 配列への即時追加
* DB 保存

これらは既存のタグ追加・保存フローに任せる。
既存のタグ正規化仕様では、カンマ分割、禁止文字除去、末尾重み除去、小文字化、重複除去、128文字制限が定義済みである。

# 9. エラー処理

## 9.1 メタ情報取得失敗

条件。

* path 未指定
* ファイル不存在
* Pillow で画像を開けない
* メタ情報読み取り中の例外

挙動。

```text
- モーダルは表示維持
- textarea は空表示
- トーストで失敗通知
```

表示文言。

```text
メタ情報を取得できませんでした
```

## 9.2 コピー時に未選択

条件。

```text
textarea 内でテキスト選択がない
```

挙動。

```text
- コピーしない
- モーダルは閉じない
- warning トーストを表示
```

表示文言。

```text
テキストが選択されていません
```

## 9.3 コピー失敗

条件。

```text
navigator.clipboard.writeText が失敗
```

挙動。

コピー操作の場合。

```text
- danger トーストを表示
- モーダルは閉じない
```

タグ付け操作の場合。

```text
- warning トーストを表示
- tagInput への反映は継続する
- モーダルを閉じる
```

理由。

```text
タグ付けの主目的は edit.vue への反映であり、クリップボードコピー失敗を理由に中断しない。
```

# 12. 注意事項

## 12.1 メタ情報は厳密な YAML ではない

本機能では、表示形式を YAML 風のプレーンテキストとする。
ただし、元画像に含まれるメタ情報形式は PNG / JPEG / WebP / AVIF 等で異なるため、厳密な YAML としての構文保証はしない。

## 12.2 タグ付けボタンで即保存しない

`タグ付け` ボタンは `edit.vue` の `tagInput` に反映するだけとする。

ここで DB 保存まで行うと、既存の `edit.vue → tile.vue → Content.vue → App.vue → update_image_file_detail` の保存経路を迂回してしまう。
その結果、保存失敗時の edit 表示維持、入力保持、再検索制御などの既存仕様と衝突する。

## 12.3 tagInput 全体の文字数制限は撤廃する

ただし、タグ単体の 128 文字制限は維持する。
巨大なメタ情報を丸ごと流し込むと画面操作が重くなる可能性があるため、将来的には警告表示や上限ガードを検討してもよい。

## 12.4 Clipboard API の環境差

pywebview 環境で `navigator.clipboard.writeText` が失敗する可能性はある。
本仕様では、コピー失敗時にトースト通知する。
必要になった場合のみ、backend 側に OS クリップボード操作 API を追加する。

今回はそこまで広げない。Clipboard まで OS API 化し始めると、機能追加というより小さな官僚機構になります。

### 8.2.2 _normalize_search_tags 追加

```python
def _normalize_search_tags(self, value) -> list[str]:
    if not isinstance(value, list):
        return []

    tags: list[str] = []

    for item in value:
        tag = str(item or "").strip()
        if not tag:
            continue
        if tag in tags:
            continue
        tags.append(tag)

    return tags[:3]
```

検索用タグはタグ検索ダイアログから選択されるため、保存時のタグ正規化処理は原則不要とする。
ただし、空文字・重複・件数超過は防御的に処理する。

### 8.2.3 fetch_tags_for_search 追加

```python
def fetch_tags_for_search(self, payload):
    try:
        data = payload or {}
        keyword = str(data.get("keyword") or "").strip()

        try:
            limit = int(data.get("limit") or 256)
        except (TypeError, ValueError):
            limit = 256

        limit = max(1, min(limit, 256))

        repository = self._get_repository()

        tags = repository.find_tags_for_search(
            keyword=keyword,
            limit=limit,
        )
        total_count = repository.count_tags_for_search(
            keyword=keyword,
        )

        return ApiResponse(
            success=True,
            message="",
            data={
                "tags": [tag.to_dict() for tag in tags],
                "total_count": total_count,
                "limit": limit,
            },
        ).to_dict()

    except Exception:
        return ApiResponse(
            success=False,
            message="タグ一覧を取得できませんでした。",
            data={
                "tags": [],
                "total_count": 0,
                "limit": 256,
            },
        ).to_dict()
```

## 14. 注意点

`fetch_tags_for_search` の `total_count` は、`LIMIT 256` 適用後の件数ではなく、現在のフィルタ条件に一致する総数とする。ここを間違えると「表示件数: 256 / 256」になり、実際には 1840 件あるのに全部見えているように見える。これは小さな数字の顔をした大きな嘘です。

タグ検索条件は `tag.name` ベースでよい。既存の一覧 item も `tags: list[str]` として扱う設計であり、表示・編集もタグ名中心の構成であるため。将来的にタグ名編集機能を追加する場合は、`tag_id` ベース検索への変更を検討する。

```python
def fetch_folders_for_search(self, payload):
    try:
        data = payload or {}
        keyword = str(data.get("keyword") or "").strip()

        try:
            limit = int(data.get("limit") or 256)
        except (TypeError, ValueError):
            limit = 256

        limit = max(1, min(limit, 256))

        repository = self._get_repository()

        folders = repository.find_folders_for_search(
            keyword=keyword,
            limit=limit,
        )
        total_count = repository.count_folders_for_search(
            keyword=keyword,
        )

        return ApiResponse(
            success=True,
            message="",
            data={
                "folders": [folder.to_dict() for folder in folders],
                "total_count": total_count,
                "limit": limit,
            },
        ).to_dict()

    except Exception:
        return ApiResponse(
            success=False,
            message="フォルダ一覧を取得できませんでした。",
            data={
                "folders": [],
                "total_count": 0,
                "limit": 256,
            },
        ).to_dict()
```

## 12. 注意点

フォルダ検索は `folder` 名の完全一致で行う。
候補一覧のフィルタは部分一致だが、画像検索時は完全一致である。

```text
候補検索:
  folder LIKE '%set%'

画像検索:
  image_file_data.folder = 'set01'
```

この違いは意図的なものとする。候補選択は探すための検索、画像検索は選んだフォルダで確定的に絞るための検索である。
ここを両方部分一致にすると、「set」で検索したつもりが `set01`, `set02`, `preset` まで混ざる可能性があり、フォルダ検索というよりフォルダ占いになります。
