### `.gitignore`
- 概要: Python、Node、ビルド成果物、仮想環境、エディタ設定、DB/データ出力などを Git 管理対象外にする定義。

### `LICENSE`
- 概要: リポジトリのライセンス文書。

### `README.md`
- 概要: AZViewer の目的、セットアップ、リリースビルド、開発時起動手順を記載するドキュメント。

### `build_windows.ps1`
- 概要: Windows 実行ファイルを作成する PowerShell スクリプト。Python 仮想環境の作成、依存関係インストール、フロントエンドビルド、PyInstaller 実行を行う。
- メソッド/関数: なし。

## backend

### `backend/__init__.py`
- 概要: `backend` パッケージを示す空の初期化ファイル。
- メソッド/関数: なし。

### `backend/main.py`
- 概要: pywebview アプリケーションのエントリポイント。Vue の表示 URL 解決、ウィンドウ生成、ドラッグ&ドロップ連携、終了時クローズを行う。
- 関数:
  - `resolve_entry_url()`: `AZVIEWER_FRONTEND_URL` または `frontend/dist/index.html` から表示 URL を解決する。
  - `create_drop_items(event)`: pywebview の drop event から `{path, type}` の登録対象一覧を作る。
  - `register_handlers(window, api)`: DOM の drop イベントに登録処理を紐付ける。
  - `register_handlers.on_drop(event)`: ドロップされたファイル/フォルダを `AppApi.import_selected_items` に渡し、完了イベントを JS 側へ通知する内部関数。
  - `main()`: アプリケーションを起動し、終了コードを返す。

### `backend/requirements.txt`
- 概要: Python 依存関係。`pywebview` と `Pillow` を指定する。

### `backend/api/app_api.py`
- 概要: pywebview に公開する最上位 API。ライフサイクル、テンプレート情報、画像登録、画像一覧操作の各 API クラスへ処理を委譲する。
- クラス:
  - `AppApi`: Vue から呼び出される API の窓口。
- メソッド:
  - `__init__()`: `ServiceManager` と各 API/サービスを組み立てる。
  - `close()`: アプリ終了時に接続を閉じる。
  - `initialize()`: 互換用初期化 API として `bootstrap_app` を呼ぶ。
  - `initialize_app()`: 基本アプリ情報を返す。
  - `bootstrap_app()`: 起動時整合性確認込みの初期化結果を返す。
  - `get_app_info()`: アプリ名やバージョンを返す。
  - `health_check()`: Python API の疎通確認結果を返す。
  - `import_selected_items(payload)`: 選択/ドロップされた画像を登録する。
  - `select_files_dialog()`: ファイル選択ダイアログを開く。
  - `select_folder_dialog()`: フォルダ選択ダイアログを開く。
  - `search_image_files(payload)`: 画像一覧を検索する。
  - `update_image_file_flags(payload)`: 確認済み/お気に入りフラグを更新する。
  - `fetchLocalImageThumb(payload)`: サムネイルの data URL を返す。
  - `_get_active_window()`: 現在の pywebview ウィンドウを返す。

### `backend/api/app_lifecycle_api.py`
- 概要: アプリ起動・終了時の処理を担当する。DB 初期化、存在しないファイルの削除、不足サムネイル生成、初期検索を行う。
- クラス:
  - `AppLifeCycleApi`: ライフサイクル API。
- メソッド:
  - `__init__(...)`: 起動時処理に必要なサービスを保持する。
  - `initialize()`: 互換用初期化 API として `bootstrap_app` を呼ぶ。
  - `bootstrap_app()`: DB 初期化、起動時整合性確認、初期検索結果をまとめて返す。
  - `close()`: DB 接続を閉じる。
  - `_run_startup_cleanup()`: 存在しない登録ファイルを削除し、通知情報を返す。
  - `_get_cleanup_service()`: 起動時クリーンアップサービスを遅延生成して返す。
  - `_ensure_thumbnail_cache()`: 既存レコードの不足サムネイルを補完する。
  - `_get_cleanup_repository()`: クリーンアップ用リポジトリを遅延生成して返す。

### `backend/api/default_template_api.py`

### `backend/api/image_register_api.py`
