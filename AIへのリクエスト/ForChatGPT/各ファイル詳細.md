## Backend

### `backend/main.py`
- 概要: pywebview アプリケーションのエントリーポイント。Vue 表示 URL の解決、ドラッグ＆ドロップ登録イベント、ウィンドウ生成、API ライフサイクル管理を行う。
- メソッド一覧:
  - `resolve_entry_url()`: 開発サーバー URL または `frontend/dist/index.html` の URL を返す。
  - `create_drop_items(event)`: drop イベントから登録用 item 配列を作る。
  - `dispatch_frontend_event(window, event_name, detail)`: Vue 側へ `CustomEvent` を送る。
  - `create_import_event_detail(result)`: 登録 API 結果をイベント detail 形式へ変換する。
  - `register_handlers(window, api)`: DOM drop イベントに登録処理を紐づける。
  - `main()`: アプリケーションを起動し、終了コードを返す。

### `backend/api/api_response.py`
- 概要: フロントエンドへ返す API レスポンスの共通 DTO。
- クラス/メソッド一覧:
  - `ApiResponse`
  - `ApiResponse.to_dict()`: `success`, `message`, `data` を持つ辞書へ変換する。

### `backend/api/app_api.py`
- 概要: pywebview から Vue に公開する統合 API。ライフサイクル、画像登録、検索、詳細更新、削除、画像取得、メタ情報取得、OS 操作を各 API へ委譲する。
- クラス/メソッド一覧:
  - `AppApi`
  - `__init__()`: API クラスと共有サービスを組み立てる。
  - `close()`: DB 接続などを閉じる。
  - `initialize()`: 互換用の初期化 API。
  - `initialize_app()`: 基本初期化情報を返す。
  - `bootstrap_app()`: 起動時整合性確認を含む初期化を行う。
  - `get_app_info()`: アプリ情報を返す。
  - `health_check()`: API 疎通確認を返す。
  - `import_selected_items(payload)`: 選択/ドロップされた画像を登録する。
  - `select_files_dialog()`: ファイル選択ダイアログを開く。
  - `select_folder_dialog()`: フォルダ選択ダイアログを開く。
  - `search_image_files(payload)`: 画像一覧を検索する。
  - `update_image_file_detail(payload)`: 画像詳細を更新する。
  - `delete_image_file(payload)`: 画像レコードを削除する。
  - `fetchLocalImage(payload)`: 元画像を data URL として返す。
  - `fetchLocalImageThumb(payload)`: サムネイルを data URL として返す。
  - `fetch_image_metadata(payload)`: 画像メタ情報取得を `ImageCatalogApi` へ委譲する。
  - `open_containing_folder(payload)`: 対象画像のフォルダを開く。
  - `_get_active_window()`: 現在の pywebview ウィンドウを返す。

### `backend/api/app_lifecycle_api.py`
- 概要: アプリ起動時の DB 初期化、登録済みファイルの存在確認、不足サムネイル生成、初期検索結果返却を担当する。
- クラス/メソッド一覧:
  - `AppLifeCycleApi`
  - `__init__(...)`: DB 管理、テンプレート API、検索 API、サムネイルサービスを保持する。
  - `initialize()`: `bootstrap_app()` を呼ぶ互換用メソッド。
  - `bootstrap_app()`: 初期化、起動時クリーンアップ、初期表示データ取得を行う。
  - `close()`: DB 接続を閉じる。
  - `_run_startup_cleanup()`: 存在しない登録ファイルを削除し、通知情報を返す。
  - `_get_cleanup_service()`: 起動時クリーンアップサービスを初期化/取得する。
  - `_ensure_thumbnail_cache()`: 既存レコードの不足サムネイルを補完する。
  - `_get_cleanup_repository()`: クリーンアップ用リポジトリを取得する。

### `backend/api/database_lifecycle_manager.py`
- 概要: アプリ全体で共有する SQLite 接続のライフサイクルを管理する。
- クラス/メソッド一覧:
  - `DatabaseLifecycleManager`
  - `__init__()`: `ConnectionManager` を準備する。
  - `initialize()`: DB 接続を初期化する。
  - `close()`: DB 接続を閉じる。
  - `get_connection()`: 初期化済み SQLite 接続を返す。

### `backend/api/default_template_api.py`
- 概要: アプリ名、バージョン、ヘルスチェックなどの基本 API を提供する。
- クラス/メソッド一覧:
  - `DefaultTemplateApi`
  - `initialize_app()`: 初期化済みアプリ情報を返す。
  - `get_app_info()`: アプリ名、バージョン、説明を返す。
  - `health_check()`: Python API の利用可否を返す。

### `backend/api/image_catalog_api.py`
- 概要: 画像一覧の検索、詳細更新、削除、元画像/サムネイル取得、メタ情報取得を提供する API。
- クラス/メソッド一覧:
  - `ImageCatalogApi`
  - `__init__(...)`: DB 管理、サムネイルサービス、メタ情報サービス、タグ正規化サービスを保持する。
  - `search_image_files(payload)`: 条件付き画像検索を行う。
  - `update_image_file_detail(payload)`: レーティング、チェック、お気に入り、コメント、タグを更新する。
  - `delete_image_file(payload)`: レコードとサムネイルを削除する。
  - `fetchLocalImage(payload)`: 元画像を base64 data URL で返す。
  - `fetchLocalImageThumb(payload)`: サムネイルを base64 data URL で返す。
  - `fetch_image_metadata(payload)`: 画像ファイルのメタ情報テキストを返す。
  - `_get_repository()`: リポジトリを初期化/取得する。
  - `_normalize_nullable_string(value)`: 空文字を `None` へ正規化する。
  - `_normalize_detail_payload(data)`: 詳細更新ペイロードを DB 更新値へ変換する。
  - `_update_detail_with_tags(detail, tags)`: 詳細とタグを同一トランザクションで更新する。
  - `_normalize_detail_comment(value)`: コメントを保存用に正規化し、長さを検証する。

### `backend/api/image_register_api.py`
- 概要: ファイル/フォルダ選択と画像登録処理を提供する API。
- クラス/メソッド一覧:

  - `__init__(...)`: DB 管理、ダイアログサービス、サムネイルサービスを保持する。
  - `_get_import_service()`: 画像登録サービスを初期化/取得する。
  - `import_selected_items(payload)`: 選択/ドロップされた item を登録する。
  - `select_files_dialog()`: ネイティブファイル選択ダイアログを開く。
  - `select_folder_dialog()`: ネイティブフォルダ選択ダイアログを開く。

### `backend/api/os_operation_api.py`

### `backend/db/connection_manager.py`

### `backend/models/image_file.py`
- 概要: 画像登録、一覧検索、インポート結果で使う dataclass DTO を定義する。
- クラス/メソッド一覧:

  - `ImageFileListItem`: 一覧表示用の画像ファイル情報。
  - `ImageFileListItem.to_dict()`: API 返却用辞書へ変換する。
  - `SearchImageFilesResult`: ページング付き検索結果。
  - `SearchImageFilesResult.to_dict()`: API 返却用辞書へ変換する。
  - `ImportResult`: 登録処理の成功/失敗と件数情報。
  - `ImportResult.to_api_data()`: フロントエンド返却用データへ変換する。

### `backend/repositories/image_file_repository.py`
- 概要: `image_file_data`, `tag`, `tag_image_link` テーブルへの CRUD、検索、タグ関連付けを担当する SQLite リポジトリ。
- クラス/メソッド一覧:
  - `ImageFileRepository`
  - `__init__(connection)`: SQLite 接続を保持する。
  - `create_table()`: 画像・タグ・リンクテーブルを作成する。
  - `find_all_paths()`: 登録済みパス一覧を取得する。
  - `find_all_items()`: 登録済み画像を全件取得する。
  - `delete_by_paths(paths)`: 指定パス群のレコードを削除し ID を返す。
  - `insert_many(records)`: 画像レコードを一括登録する。
  - `exists_by_path(path)`: パスの登録有無を判定する。
  - `find_by_paths(paths)`: 指定パス群の一覧項目を取得する。
  - `find_ids_by_paths(paths)`: 指定パス群の ID を取得する。
  - `search(...)`: 条件、ページング、ソート付きで検索する。
  - `delete_by_id(record_id)`: ID 指定で削除する。
  - `update_detail(...)`: 詳細項目を更新する。
  - `find_tags_by_image_ids(image_ids)`: 画像 ID ごとのタグ一覧を取得する。
  - `replace_tags(image_file_id, tags)`: 画像に紐づくタグを置換する。
  - `find_by_id(record_id)`: ID 指定で一覧項目を取得する。
  - `_row_to_item(row)`: SQLite 行を DTO へ変換する。
  - `_attach_tags(items)`: 一覧項目にタグを付与する。
  - `_find_or_create_tag_id(tag_name)`: タグ ID を取得または作成する。
  - `_build_order_by(sort)`: 許可済み sort から SQL の ORDER BY 句を返す。
  - `_normalize_true_condition(value)`: 検索条件の真偽値を正規化する。
  - `_validate_detail_values(...)`: 詳細更新値を検証する。

### `backend/services/dialog_service.py`
- 概要: pywebview のネイティブダイアログを使い、画像ファイルまたはフォルダを選択する。
- クラス/メソッド一覧:
  - `DialogService`
  - `__init__(window_provider)`: ダイアログ表示用ウィンドウ取得関数を保持する。
  - `select_files()`: 複数画像ファイル選択結果を item 配列で返す。
  - `select_folder()`: フォルダ選択結果を item 配列で返す。
  - `_get_window()`: 利用可能な pywebview ウィンドウを取得する。

### `backend/services/file_scan_service.py`
- 概要: 登録対象 item から対応画像ファイルを抽出し、DB 登録用レコードを作る。
- クラス/メソッド一覧:
  - `FileScanService`
  - `is_supported_image(path)`: 対応拡張子の画像ファイルか判定する。
  - `collect_image_paths(items)`: ファイル/フォルダ item から画像パスを収集する。
  - `build_record(path)`: 画像パスから `ImageFileRecord` を作る。
  - `_collect_direct_children(directory)`: フォルダ直下の対応画像を収集する。
  - `_dedupe_paths(paths)`: 順序を維持して重複パスを除去する。

### `backend/services/image_file_import_service.py`
- 概要: 選択/ドロップされたファイルやフォルダから未登録画像を DB へ登録し、サムネイル生成を行う。
- クラス/メソッド一覧:
  - `ImageFileImportService`
  - `__init__(...)`: DB 接続、リポジトリ、スキャン、サムネイルサービスを保持する。
  - `import_items(payload)`: 入力を検証し、未登録画像を登録して結果を返す。

### `backend/services/image_metadata_service.py`
- 概要: Pillow で画像ファイルを開き、基本情報と画像内メタ情報を YAML 風プレーンテキストへ整形する。
- クラス/メソッド一覧:
  - `ImageMetadataService`
  - `fetch_metadata_text(path)`: ファイル存在を検証し、画像メタ情報テキストを返す。
  - `_build_metadata_text(image_path, image)`: ファイル名、パス、形式、サイズ、`image.info` を行リストへ変換する。
  - `_format_metadata_value(key, value)`: 単一メタ値を 1 行または複数行ブロック形式へ整形する。
  - `_to_text(value)`: bytes を UTF-8 文字列へ変換し、それ以外は文字列化する。

### `backend/services/startup_cleanup_service.py`
- 概要: 起動時に DB 登録済み画像の実ファイル存在確認を行い、存在しないものを削除する。
- クラス/メソッド一覧:
  - `StartupCleanupService`
  - `__init__(...)`: DB 接続、リポジトリ、サムネイルサービスを保持する。
  - `cleanup_missing_files()`: 実ファイルがないレコードとサムネイルを削除し、削除件数を返す。

### `backend/services/tag_normalize_service.py`
- 概要: タグ入力値の分割、非エスケープ括弧除去、重み表記除去、小文字化、重複除去、長さ検証を行う。
- クラス/メソッド一覧:
  - `TagNormalizeService`
  - `normalize_tags(values)`: 入力値を保存用タグ一覧へ正規化する。
  - `_normalize_tag_values(value)`: 1入力値をカンマ分割し、タグ配列へ変換する。
  - `_normalize_tag_text(value)`: タグ文字列を保存用表記へ正規化する。
  - `_remove_unescaped_brackets(value)`: バックスラッシュでエスケープされていない `()[]{}` だけ除去する。

### `backend/services/thumbnail_cache_service.py`
- 概要: 画像ファイルから 300px サムネイル PNG を生成し、取得・削除する。
- クラス/メソッド一覧:
  - `ThumbnailCacheService`
  - `__init__(cache_dir=None)`: サムネイル保存先を準備する。
  - `ensure_thumbnails(items, missing_only=True)`: 一覧項目に対応するサムネイルを生成する。
  - `create_thumbnail(record_id, source_path)`: 元画像からサムネイルを作成する。
  - `delete_thumbnails(record_ids)`: 複数サムネイルを削除する。
  - `delete_thumbnail(record_id)`: 1件のサムネイルを削除する。
  - `get_thumbnail_bytes(record_id)`: サムネイルのバイト列を返す。
  - `get_thumb_path(record_id)`: サムネイル保存パスを返す。
