## Backend

### `backend/main.py`
- 概要: pywebview アプリケーションのエントリーポイント。Vue 表示 URL の解決、ドラッグ＆ドロップ登録イベント、ウィンドウ生成、API ライフサイクル管理を行う。
- メソッド一覧:
  - `resolve_entry_url()`: 開発サーバー URL または `frontend/dist/index.html` の URL を返す。
  - `create_drop_items(event)`: drop イベントから登録用 item 配列を作る。
  - `dispatch_frontend_event(window, event_name, detail)`: Vue 側へ `CustomEvent` を送る。
  - `create_import_event_detail(result)`: 登録 API 結果をイベント detail 形式へ変換する。
  - `register_handlers(window, api)`: DOM drop イベントに登録処理を紐づける。
  - `main()`: アプリケーションを起動し、終了コードを返す。

### `backend/api/api_response.py`
- 概要: フロントエンドへ返す API レスポンスの共通 DTO。
- クラス/メソッド一覧:
  - `ApiResponse`: API 共通レスポンス。
  - `ApiResponse.to_dict()`: `success`, `message`, `data` を持つ辞書へ変換する。

- クラス/メソッド一覧:
  - `AppApi`: pywebview 公開 API の集約クラス。
  - `__init__()`: API クラスと共有サービスを組み立てる。
  - `close()`: DB 接続などを閉じる。
  - `initialize()`: 互換用の初期化 API。
  - `initialize_app()`: 基本初期化情報を返す。
  - `bootstrap_app()`: 起動時整合性確認を含む初期化を行う。
  - `get_app_info()`: アプリ情報を返す。
  - `health_check()`: API 疎通確認を返す。
  - `import_selected_items(payload)`: 選択/ドロップされた画像を登録する。
  - `select_files_dialog()`: ファイル選択ダイアログを開く。
  - `select_folder_dialog()`: フォルダ選択ダイアログを開く。
  - `search_image_files(payload)`: 画像一覧を検索する。
  - `update_image_file_detail(payload)`: 画像詳細を更新する。
  - `delete_image_file(payload)`: 画像レコードを削除する。
  - `delete_image_files_with_physical_files(payload)`: 指定画像の実ファイルを含む一括削除を `ImageCatalogApi` へ委譲する。

  - `fetchLocalImage(payload)`: 元画像を data URL として返す。
  - `fetchLocalImageThumb(payload)`: サムネイルを data URL として返す。
  - `fetch_image_metadata(payload)`: 画像メタ情報取得を `ImageCatalogApi` へ委譲する。
  - `fetch_tags_for_search(payload)`: タグ検索候補取得を `ImageCatalogApi` へ委譲する。
  - `fetch_folders_for_search(payload)`: フォルダ検索候補取得を `ImageCatalogApi` へ委譲する。
  - `fetch_models_for_search(payload)`: モデル検索候補取得を `ImageCatalogApi` へ委譲する。
  - `fetch_duplicate_tag_sets(payload=None)`: 重複タグ構成一覧取得を `ImageCatalogApi` へ委譲する。
  - `import_prompt_tags(payload=None)`: プロンプト一括タグ登録を `ImageCatalogApi` へ委譲する。
  - `export_wildcard_text(payload=None)`: Stable Diffusion WebUI ワイルドカード用テキストの保存/追記を `ImageCatalogApi` へ委譲する。
  - `open_containing_folder(payload)`: 対象画像のフォルダを開く。
  - `_get_active_window()`: 現在の pywebview ウィンドウを返す。

### `backend/api/image_catalog_api.py`
- 概要: 画像一覧の検索、詳細更新、単体削除、一括物理削除、一括ファイル移動、ワイルドカード出力、元画像/サムネイル取得、メタ情報取得、タグ/フォルダ/モデル検索候補取得、重複タグ構成取得、プロンプト・モデル一括読み取りを提供する API。
- クラス/メソッド一覧:
  - `ImageCatalogApi`: 画像カタログ操作 API。
  - `__init__(...)`: DB 管理、サムネイルサービス、メタ情報サービス、タグ正規化サービス、共有ダイアログサービスを保持する。
  - `search_image_files(payload)`: 条件付き画像検索を行う。`model` 指定時はモデル名完全一致条件を追加し、`tag_hash` と `tag_set` が両方ある場合は重複タグ構成条件も適用する。
  - `update_image_file_detail(payload)`: レーティング、チェック、お気に入り、コメント、タグ、生成元モデルを同一トランザクションで更新する。
  - `delete_image_file(payload)`: レコードとサムネイルを削除する。
  - `delete_image_files_with_physical_files(payload)`: ID 配列を受け取り、実ファイル、サムネイル、DB レコードを一括削除する。
  - `move_image_files_to_folder(payload)`: ID 配列と移動先フォルダを受け取り、実ファイルを移動して成功分の DB `path` / `folder` を更新する。
  - `fetchLocalImage(payload)`: 元画像を base64 data URL で返す。
  - `fetchLocalImageThumb(payload)`: サムネイルを base64 data URL で返す。
  - `fetch_image_metadata(payload)`: 画像ファイルのメタ情報テキストを返す。
  - `fetch_tags_for_search(payload)`: タグ検索ダイアログ向け候補と総件数を返す。
  - `fetch_folders_for_search(payload)`: フォルダ検索ダイアログ向け候補と総件数を返す。
  - `fetch_models_for_search(payload)`: モデル検索ダイアログ向け候補と総件数を返す。
  - `fetch_duplicate_tag_sets(payload=None)`: 重複しているタグ構成を件数付きで返す。
  - `import_prompt_tags(payload=None)`: タグ未登録画像へ Stable Diffusion WebUI プロンプト由来タグを登録し、モデル未設定画像へ生成元モデル名を一括登録する。
  - `export_wildcard_text(payload=None)`: フロントエンドで確定したプレビューテキストを `.txt` として新規保存または既存ファイルへ追記する。
  - `_get_repository()`: リポジトリを初期化/取得する。
  - `_get_wildcard_export_service()`: 共有 `DialogService` を使ってワイルドカード出力サービスを初期化/取得する。
  - `_normalize_nullable_string(value)`: 空文字を `None` へ正規化する。
  - `_normalize_search_tags(value)`: 検索条件用タグを空文字・重複除去し最大3件へ丸める。
  - `_normalize_search_folder(value)`: 検索条件用フォルダを空文字なら `None` へ正規化する。
  - `_normalize_search_model(value)`: 検索条件用モデル名を空文字なら `None` へ正規化する。
  - `_normalize_tag_hash_condition(data)`: `tag_hash` / `tag_set` の両方が揃う場合だけ検索条件として正規化する。
  - `_normalize_tag_search_limit(value)`: 候補取得件数を1から256の範囲へ丸める。
  - `_normalize_model_search_limit(value)`: モデル候補取得件数を1から256の範囲へ丸める。
  - `_normalize_detail_payload(data)`: 詳細更新ペイロードを DB 更新値へ変換する。
  - `_normalize_model_name(value)`: 詳細保存用モデル名を trim し、空文字は未設定、512文字超過はエラーにする。
  - `_normalize_delete_ids(value)`: 一括削除対象 ID を正の整数の重複なしリストへ正規化する。
  - `_normalize_record_ids(value)`: レコード ID 配列を正の整数の重複なしリストへ正規化する。
  - `_normalize_destination_folder(value)`: 移動先フォルダを絶対パス化し、存在するディレクトリか検証する。
  - `_empty_physical_delete_data()`: 一括物理削除 API 用の空集計データを返す。
  - `_empty_file_move_data()`: 一括ファイル移動 API 用の空集計データを返す。
  - `_empty_wildcard_export_data()`: ワイルドカード出力 API 用の空結果データを返す。
  - `_delete_physical_files_and_records(repository, items)`: 実ファイル削除に成功または実ファイルが不存在の画像だけ、サムネイルと DB レコードを削除する。
  - `_delete_physical_files(items)`: 実ファイル削除を1件ずつ試行し、削除可能 ID と失敗情報を集計する。
  - `_delete_physical_file(path)`: 1つの実ファイルを `Path.unlink()` で削除し、削除済み/不存在を返す。
  - `_move_files_and_update_records(repository, items, destination_folder)`: 移動可能な実ファイルを移動し、成功分だけ DB パスを更新する。
  - `_move_files(repository, items, destination_folder)`: 各画像の移動を試行し、移動済み候補、スキップ数、失敗情報を集計する。
  - `_move_one_file(repository, item, destination_folder)`: 1画像ファイルを移動し、DB 更新候補を返す。同一パスの場合はスキップ扱いにする。
  - `_build_path_updates(moved_items)`: DB の `path` / `folder` 更新用 payload を作る。
  - `_restore_moved_files(moved_items)`: DB 更新失敗時に移動済みファイルを元の場所へ戻す。
  - `_append_file_move_failure(failed_files, record_id, path, error)`: ファイル移動失敗情報を最大20件まで追加する。
  - `_append_file_move_message(failed_files, record_id, path, reason)`: ファイル移動失敗メッセージを最大20件まで追加する。
  - `_to_file_move_failure_reason(error)`: ファイル移動例外をユーザー向け理由へ変換する。
  - `_update_detail_with_tags_and_model(detail, tags, model_name)`: 詳細、タグ、モデルを同一トランザクションで更新する。
  - `_normalize_detail_comment(value)`: コメントを保存用に正規化し、長さを検証する。

### `backend/repositories/image_file_repository.py`
