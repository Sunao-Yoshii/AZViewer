## Backend

### `backend/api/api_response.py`
- 概要: pywebview API から Vue へ返す共通レスポンス形式を定義する。
- クラス/メソッド一覧:
  - `ApiResponse.to_dict()`: `success`、`message`、`data` を辞書形式へ変換する。

### `backend/api/app_api.py`
- 概要: pywebview を通じて Vue へ公開する最上位 API。ライフサイクル、基本情報、画像登録、画像一覧操作へ処理を委譲する。
- クラス/メソッド一覧:
  - `AppApi.__init__()`: DB 接続ライフサイクル管理と各 API/サービスを組み立てる。
  - `AppApi.close()`: アプリ終了時に共有接続を閉じる。
  - `AppApi.initialize()`: 互換用の初期化 API として `AppLifeCycleApi.initialize()` を呼ぶ。
  - `AppApi.initialize_app()`: Vue 起動時の基本初期化情報を返す。
  - `AppApi.bootstrap_app()`: 起動時整合性確認を含む初期化結果を返す。
  - `AppApi.get_app_info()`: アプリケーション名やバージョンを返す。
  - `AppApi.health_check()`: Python API の疎通確認結果を返す。
  - `AppApi.import_selected_items(payload)`: 選択またはドロップされた画像を登録する。
  - `AppApi.select_files_dialog()`: ネイティブファイル選択ダイアログを開く。
  - `AppApi.select_folder_dialog()`: ネイティブフォルダ選択ダイアログを開く。
  - `AppApi.search_image_files(payload)`: 画像一覧を検索する。
  - `AppApi.update_image_file_detail(payload)`: 指定レコードの詳細項目を一括更新する。
  - `AppApi.delete_image_file(payload)`: 指定レコードを削除する。
  - `AppApi.fetchLocalImage(payload)`: ローカル画像本体を data URL として返す。
  - `AppApi.fetchLocalImageThumb(payload)`: サムネイルを data URL として返す。
  - `AppApi._get_active_window()`: 現在利用可能な pywebview ウィンドウを返す。

### `backend/api/app_lifecycle_api.py`
- 概要: アプリケーション起動・終了時の処理を担当する。DB 初期化、起動時クリーンアップ、サムネイル補完、初期検索を行う。
- クラス/メソッド一覧:
  - `AppLifeCycleApi.__init__(...)`: DB 接続管理、基本 API、画像一覧 API、サムネイルサービスを保持する。
  - `AppLifeCycleApi.initialize()`: 互換用に `bootstrap_app()` を呼び出す。
  - `AppLifeCycleApi.bootstrap_app()`: 初期化、起動時整合性確認、初期検索結果作成を行う。
  - `AppLifeCycleApi.close()`: 共有 DB 接続を閉じる。
  - `AppLifeCycleApi._run_startup_cleanup()`: 存在しない登録ファイルを削除し、通知情報を返す。
  - `AppLifeCycleApi._get_cleanup_service()`: 起動時クリーンアップサービスを遅延初期化して返す。
  - `AppLifeCycleApi._ensure_thumbnail_cache()`: 既存レコードの不足サムネイルを補完する。
  - `AppLifeCycleApi._get_cleanup_repository()`: クリーンアップ用リポジトリを返す。

### `backend/api/database_lifecycle_manager.py`
- 概要: アプリケーション全体で共有する SQLite 接続のライフサイクルのみを管理する。Repository / Service / API インスタンスは保持しない。
- クラス/メソッド一覧:
  - `DatabaseLifecycleManager.__init__()`: 共有 SQLite 接続管理用の `ConnectionManager` を準備する。
  - `DatabaseLifecycleManager.initialize()`: 共有 SQLite 接続を初期化する。
  - `DatabaseLifecycleManager.close()`: 共有 SQLite 接続を閉じる。
  - `DatabaseLifecycleManager.get_connection()`: 初期化済みの共有 SQLite 接続を返す。

### `backend/api/default_template_api.py`
- 概要: アプリ情報、初期化情報、ヘルスチェックなどの基本 API を提供する。
- クラス/メソッド一覧:
  - `DefaultTemplateApi.initialize_app()`: アプリ名と初期化時刻を返す。
  - `DefaultTemplateApi.get_app_info()`: アプリ名、バージョン、説明を返す。
  - `DefaultTemplateApi.health_check()`: Python API が利用可能であることを示す応答を返す。

### `backend/api/image_catalog_api.py`
- 概要: 画像一覧検索、詳細更新、レコード削除、画像本体取得、サムネイル取得を提供する。
- クラス/メソッド一覧:
  - `ImageCatalogApi.__init__(...)`: DB 接続管理とサムネイルサービスを保持する。
  - `ImageCatalogApi.search_image_files(payload)`: 検索条件を正規化して画像一覧をページング取得する。
  - `ImageCatalogApi.update_image_file_detail(payload)`: rating、チェック、お気に入り、コメントを一括更新する。
  - `ImageCatalogApi.delete_image_file(payload)`: 指定レコードと対応サムネイルを削除する。
  - `ImageCatalogApi.fetchLocalImage(payload)`: path 指定で画像本体を読み込み data URL として返す。
  - `ImageCatalogApi.fetchLocalImageThumb(payload)`: ID 指定でサムネイルを data URL として返す。
  - `ImageCatalogApi._get_repository()`: 画像一覧用リポジトリを遅延初期化して返す。
  - `ImageCatalogApi._normalize_nullable_string(value)`: 空文字列を `None` へ正規化する。
  - `ImageCatalogApi._normalize_detail_payload(data)`: 詳細更新 payload を repository 用の値へ正規化する。

### `backend/api/image_register_api.py`
- 概要: 画像登録とファイル/フォルダ選択ダイアログを提供する。
- クラス/メソッド一覧:
  - `ImageRegisterApi.__init__(...)`: DB 接続管理、ダイアログサービス、サムネイルサービスを保持する。
  - `ImageRegisterApi._get_import_service()`: 画像登録サービスを遅延初期化して返す。
  - `ImageRegisterApi.import_selected_items(payload)`: 選択またはドロップされたファイル情報を画像データとして登録する。
  - `ImageRegisterApi.select_files_dialog()`: ネイティブファイル選択ダイアログを開く。
  - `ImageRegisterApi.select_folder_dialog()`: ネイティブフォルダ選択ダイアログを開く。

### `backend/db/connection_manager.py`
- 概要: SQLite 接続の実体を管理する。DB ファイル作成、接続初期化、接続取得、クローズを担当する。
- クラス/メソッド一覧:
  - `ConnectionManager.__init__(db_path=None)`: DB ファイルパスと接続状態を初期化する。
  - `ConnectionManager.db_path`: DB ファイルパスを返すプロパティ。
  - `ConnectionManager.initialize()`: SQLite 接続を作成し、row factory と foreign key 設定を適用する。
  - `ConnectionManager.get_connection()`: 初期化済み SQLite 接続を返す。
  - `ConnectionManager.close()`: 保持している SQLite 接続を閉じる。

### `backend/main.py`
- 概要: pywebview アプリケーションのエントリポイント。表示 URL 解決、ウィンドウ生成、ドラッグ＆ドロップ連携、終了時クローズを行う。
- 関数一覧:
  - `resolve_entry_url()`: `AZVIEWER_FRONTEND_URL` または `frontend/dist/index.html` から表示 URL を解決する。
  - `create_drop_items(event)`: pywebview drop event から登録対象 `{path, type}` の一覧を作る。
  - `register_handlers(window, api)`: DOM の drop イベントへ登録処理を紐づける。
  - `register_handlers.on_drop(event)`: ドロップされたファイル/フォルダを登録し、完了イベントを JS 側へ通知する内部関数。
  - `main()`: アプリケーションを起動し、終了コードを返す。

### `backend/models/image_file.py`
- 概要: 画像ファイル登録、一覧表示、検索結果、インポート結果の DTO を定義する。
- クラス/メソッド一覧:

  - `ImageFileListItem.to_dict()`: 一覧項目を API 返却用辞書へ変換する。
  - `SearchImageFilesResult.to_dict()`: 検索結果を API 返却用辞書へ変換する。
  - `ImportResult.to_api_data()`: インポート結果を Vue 向けレスポンス data へ変換する。

### `backend/repositories/image_file_repository.py`
- 概要: `image_file_data` テーブルへの作成、検索、登録、削除、詳細更新を担当する SQLite リポジトリ。
- クラス/メソッド一覧:
  - `ImageFileRepository.__init__(connection)`: SQLite 接続を保持する。
  - `ImageFileRepository.create_table()`: 画像情報テーブルを作成する。
  - `ImageFileRepository.find_all_paths()`: 全登録パスを取得する。
  - `ImageFileRepository.find_all_items()`: 全登録項目を取得する。
  - `ImageFileRepository.delete_by_paths(paths)`: 指定パスのレコードを削除し、削除 ID を返す。
  - `ImageFileRepository.insert_many(records)`: 複数レコードを一括登録する。
  - `ImageFileRepository.exists_by_path(path)`: 指定パスの登録有無を判定する。
  - `ImageFileRepository.find_by_paths(paths)`: 指定パス群の項目を取得する。
  - `ImageFileRepository.find_ids_by_paths(paths)`: 指定パス群の ID を取得する。
  - `ImageFileRepository.search(...)`: 条件検索とページングを行う。
  - `ImageFileRepository.delete_by_id(record_id)`: ID に一致するレコードを削除する。
  - `ImageFileRepository.update_detail(...)`: 詳細項目を一括更新する。
  - `ImageFileRepository.find_by_id(record_id)`: ID で 1 件取得する。
  - `ImageFileRepository._row_to_item(row)`: SQLite 行を一覧表示 DTO へ変換する。
  - `ImageFileRepository._build_order_by(sort)`: 許可済みソート値から `ORDER BY` 句を生成する。
  - `ImageFileRepository._normalize_true_condition(value)`: 検索条件値を true 条件として評価する。
  - `ImageFileRepository._validate_detail_values(rating, is_checked, is_favorite)`: 詳細更新で許可する値か検証する。

### `frontend/src/App.vue`
- Vue 部品: アプリケーションのルートコンポーネント。

  - `handleSortChange(sort)`: 並び順変更時に検索する。
  - `handleOpenDetail(item)`: 詳細モーダルを開き、画像本体を取得する。
  - `handleCloseDetail()`: 詳細モーダルを閉じ、詳細画像状態を初期化する。
  - `handleDelete(id)`: confirm 後に削除 API を呼び、成功時に再検索する。
  - `handleSaveDetail(payload)`: 詳細更新 API を呼び、成功時にモーダルを閉じて再検索する。
  - `handleImportedEvent()`: pywebview drop 登録完了イベントを処理する。
