## バックエンド

### `backend/main.py`
- 概要: pywebview アプリケーションのエントリポイント。Vue の表示 URL を解決し、ウィンドウ作成、ドラッグ＆ドロップ登録イベント、アプリ終了時の API クローズを管理する。
- メソッド一覧:
  - `resolve_entry_url()`: `AZVIEWER_FRONTEND_URL` または `frontend/dist/index.html` から起動 URL を決定する。
  - `create_drop_items(event)`: pywebview の drop event から登録対象の file/directory 情報を生成する。
  - `register_handlers(window, api)`: DOM drop イベントを pywebview に登録し、ドロップされた項目を import API へ渡す。
  - `main()`: アプリ起動処理を実行し、終了コードを返す。
  - `on_drop(event)`: `register_handlers` 内のローカル関数。ドロップされたファイルを登録し、成功時に `azviewer:import-complete` を dispatch する。

### `backend/api/api_response.py`
- 概要: Vue 側へ返す pywebview API レスポンスの共通 DTO。
- クラス一覧:
  - `ApiResponse`: `success`、`message`、`data` を持つ dataclass。
- メソッド一覧:
  - `ApiResponse.to_dict()`: API 返却用の辞書へ変換する。

### `backend/api/app_api.py`
- 概要: pywebview に公開する統合 API。ライフサイクル、テンプレート、画像一覧、画像登録 API へ処理を委譲する窓口。
- クラス一覧:
  - `AppApi`: フロントエンドから呼び出される API メソッド群を提供する。
- メソッド一覧:
  - `__init__()`: DB 管理、基本 API、サムネイルサービス、画像カタログ/登録 API を組み立てる。
  - `close()`: アプリ終了時に内部接続を閉じる。
  - `initialize()`: 互換用初期化 API。`bootstrap_app` へ委譲する。
  - `initialize_app()`: 基本初期化情報を返す。
  - `bootstrap_app()`: 起動時整合性確認を含む初期化結果を返す。
  - `get_app_info()`: アプリ名やバージョンを返す。
  - `health_check()`: Python API の疎通確認結果を返す。
  - `import_selected_items(payload)`: 選択/ドロップされた項目を画像データとして登録する。
  - `select_files_dialog()`: ファイル選択ダイアログを開く。
  - `select_folder_dialog()`: フォルダ選択ダイアログを開く。
  - `search_image_files(payload)`: 画像一覧検索を行う。
  - `update_image_file_detail(payload)`: 指定画像の詳細情報を更新する。
  - `delete_image_file(payload)`: 指定画像レコードを削除する。
  - `fetchLocalImage(payload)`: 画像本体を data URL で返す。
  - `fetchLocalImageThumb(payload)`: サムネイル画像を data URL で返す。
  - `_get_active_window()`: 現在利用可能な pywebview window を返す。
