## Backend

### `backend/api/api_response.py`
- 概要: pywebview 経由で Vue に返す API レスポンスの共通 DTO を定義する。
- クラス/メソッド一覧:
  - `ApiResponse.to_dict()`: `success`、`message`、`data` を辞書形式へ変換する。

### `backend/api/app_api.py`
- 概要: Vue 側へ公開する pywebview API の集約クラス。ライフサイクル、テンプレート情報、画像登録、画像検索・更新 API へ処理を委譲する。
- クラス/メソッド一覧:
  - `AppApi.__init__()`: 内部 API とサービスを組み立てる。
  - `AppApi.close()`: アプリ終了時のリソース解放を行う。
  - `AppApi.initialize()`: 互換用の初期化 API。
  - `AppApi.initialize_app()`: Vue 起動時の基本初期化情報を返す。
  - `AppApi.bootstrap_app()`: 起動整合性確認を含む初期化結果を返す。
  - `AppApi.get_app_info()`: アプリ名やバージョンを返す。
  - `AppApi.health_check()`: Python API の疎通確認結果を返す。
  - `AppApi.import_selected_items(payload)`: 選択・ドロップされた画像を登録する。
  - `AppApi.select_files_dialog()`: ファイル選択ダイアログを開く。
  - `AppApi.select_folder_dialog()`: フォルダ選択ダイアログを開く。
  - `AppApi.search_image_files(payload)`: 画像一覧を検索する。
  - `AppApi.fetchLocalImageThumb(payload)`: サムネイルの data URL を返す。
  - `AppApi._get_active_window()`: 現在の pywebview ウィンドウを取得する。

### `backend/api/app_lifecycle_api.py`
- 概要: アプリケーション起動・終了に関わる処理を担当する。DB 初期化、起動時クリーンアップ、サムネイル補完、初期検索結果の作成を行う。
- クラス/メソッド一覧:
  - `AppLifeCycleApi.__init__(...)`: 起動処理に必要なサービスを保持する。
  - `AppLifeCycleApi.initialize()`: 互換用に `bootstrap_app()` を呼び出す。
  - `AppLifeCycleApi.bootstrap_app()`: 初期化、起動時整合性確認、初期検索を実行する。
  - `AppLifeCycleApi.close()`: サービス管理側の接続を閉じる。
  - `AppLifeCycleApi._run_startup_cleanup()`: 存在しないファイルの登録情報を削除し、通知情報を返す。
  - `AppLifeCycleApi._get_cleanup_service()`: 起動時クリーンアップサービスを取得する。
  - `AppLifeCycleApi._ensure_thumbnail_cache()`: 既存レコードの不足サムネイルを補完する。
  - `AppLifeCycleApi._get_cleanup_repository()`: クリーンアップ用リポジトリを取得する。

### `backend/api/default_template_api.py`
- 概要: アプリ名、バージョン、初期化時刻、ヘルスチェックなどの基本 API を提供する。
- クラス/メソッド一覧:
  - `DefaultTemplateApi.initialize_app()`: 初期化済みアプリ情報を返す。
  - `DefaultTemplateApi.get_app_info()`: アプリケーション情報を返す。
  - `DefaultTemplateApi.health_check()`: API 疎通確認用の応答を返す。

### `backend/api/image_catalog_api.py`
- 概要: 画像一覧の検索、フラグ更新、サムネイル取得を扱う API。
- クラス/メソッド一覧:
  - `ImageCatalogApi.__init__(...)`: サービス管理とサムネイルサービスを保持する。
  - `ImageCatalogApi.search_image_files(payload)`: 検索条件を正規化して一覧を取得する。
  - `ImageCatalogApi.fetchLocalImageThumb(payload)`: サムネイル PNG を base64 data URL として返す。
  - `ImageCatalogApi._get_repository()`: 一覧用リポジトリを遅延初期化する。
  - `ImageCatalogApi._normalize_nullable_string(value)`: 空文字列を `None` に正規化する。

### `backend/api/image_register_api.py`
- 概要: 画像登録系の API。ネイティブダイアログによる選択と、選択項目のインポートを扱う。
- クラス/メソッド一覧:
  - `ImageRegisterApi.__init__(...)`: サービス管理、ダイアログ、サムネイルサービスを保持する。
  - `ImageRegisterApi._get_import_service()`: 画像登録サービスを遅延初期化する。

### `backend/api/database_lifecycle_manager.py`
- 概要: アプリケーション全体で共有する SQLite 接続のライフサイクルのみを管理する。Repository / Service / API インスタンスは保持しない。
- クラス/メソッド一覧:
  - `DatabaseLifecycleManager.__init__()`: 共有 SQLite 接続の管理に使う `ConnectionManager` を準備する。
  - `DatabaseLifecycleManager.initialize()`: 共有 SQLite 接続を初期化する。
  - `DatabaseLifecycleManager.close()`: 共有 SQLite 接続を閉じる。
  - `DatabaseLifecycleManager.get_connection()`: 初期化済みの共有 SQLite 接続を返す。

### `backend/db/connection_manager.py`
- 概要: アプリケーション全体で利用する SQLite 接続を管理する。
- クラス/メソッド一覧:
  - `ConnectionManager.__init__(db_path=None)`: DB パスと接続状態を初期化する。
  - `ConnectionManager.db_path`: DB ファイルパスを返すプロパティ。
  - `ConnectionManager.initialize()`: DB ディレクトリを作成し SQLite 接続を開く。
  - `ConnectionManager.get_connection()`: 初期化済み接続を返す。
  - `ConnectionManager.close()`: 接続を閉じる。

### `backend/main.py`
- 概要: pywebview アプリケーションのエントリポイント。表示 URL の解決、ウィンドウ作成、ドラッグ＆ドロップイベントの登録、終了時の API クローズを行う。
- 関数一覧:
  - `resolve_entry_url()`: Vite dev server URL または `frontend/dist/index.html` を解決する。
  - `create_drop_items(event)`: pywebview の drop イベントから登録用 item 配列を作る。
  - `register_handlers(window, api)`: DOM の drop イベントへ登録処理を紐づける。
  - `on_drop(event)`: `register_handlers` 内のネスト関数。ドロップされた項目を登録し、完了イベントを Vue 側へ通知する。
  - `main()`: アプリを起動し終了コードを返す。

### `backend/models/image_file.py`

  - `ImageFileRepository.find_all_paths()`: 全登録パスを取得する。
  - `ImageFileRepository.find_all_items()`: 全登録項目を取得する。

  - `ImageFileRepository.insert_many(records)`: 複数レコードを一括登録する。
  - `ImageFileRepository.exists_by_path(path)`: 指定パスの登録有無を判定する。
  - `ImageFileRepository.find_by_paths(paths)`: 指定パス群の項目を取得する。
  - `ImageFileRepository.find_ids_by_paths(paths)`: 指定パス群の ID を取得する。
  - `ImageFileRepository.search(...)`: 条件検索とページングを行う。
  - `ImageFileRepository.find_by_id(record_id)`: ID で 1 件取得する。
  - `ImageFileRepository._row_to_item(row)`: SQLite 行を DTO へ変換する。

- 概要: サムネイル、ファイル名、フォルダ、レーティング、ID、チェック/お気に入りスイッチ、コメントを表示し、フラグ変更を通知する。
- 関数一覧:
  - `loadThumbnail(id)`: backend API からサムネイル data URL を取得する。
  - `toggleFlag(field, event)`: チェック/お気に入りの変更内容を `update-flag` イベントで送出する。

### `frontend/src/components/upload/ErrorModal.vue`
- Vue 部品: 画像登録エラー表示用モーダルコンポーネント。
- 概要: エラー概要、詳細、登録失敗ファイル一覧を表示し、閉じるイベントを送出する。
- メソッド一覧: なし。

### `frontend/src/components/upload/FileSelectArea.vue`
- Vue 部品: 画像ファイル/フォルダ登録操作コンポーネント。
- 概要: ドラッグ＆ドロップ領域、ファイル選択ボタン、フォルダ選択ボタン、登録状態、エラーモーダルを提供する。
- 関数一覧:
  - `closeErrorModal()`: エラーモーダル状態を初期化して閉じる。
  - `importItems(items)`: 選択項目を backend API に渡して登録する。
  - `handleSelectFiles()`: ファイル選択ダイアログを開いて登録する。
  - `handleSelectFolder()`: フォルダ選択ダイアログを開いて登録する。
  - `handleDragEvent(event)`: ドラッグ/ドロップイベント情報をログ出力する。

### `frontend/src/main.js`
- 概要: Vue アプリの frontend エントリポイント。Bootstrap とアプリ CSS を読み込み、ブラウザ既定のファイルドロップ挙動を抑止し、`App.vue` をマウントする。
- 関数一覧:
  - `preventFileDropDefaults(event)`: drag/drop の既定動作を抑止し、drop effect を copy にする。

### `frontend/src/services/backendApi.js`
- 概要: pywebview API 呼び出しをラップする frontend 側サービス。API 利用不可時や例外時の共通レスポンス化も行う。
- 関数一覧:
  - `createUnavailableResponse(methodName)`: API 未利用時の失敗レスポンスを作る。
  - `waitForPywebviewApi(timeoutMs)`: `window.pywebview.api` の準備完了を待つ。
  - `callBackendApi(methodName, ...args)`: 指定名の backend API を安全に呼び出す。
  - `importSelectedItems(items)`: 画像登録 API を呼ぶ。
  - `selectFilesDialog()`: ファイル選択 API を呼ぶ。
  - `selectFolderDialog()`: フォルダ選択 API を呼ぶ。
  - `searchImageFiles(payload)`: 画像検索 API を呼ぶ。
  - `fetchLocalImageThumb(id)`: サムネイル取得 API を呼ぶ。

### `frontend/src/styles/app.css`
- 概要: アプリ全体のレイアウト、サイドバー、ファイル選択領域、画像グリッド、画像タイル、起動 overlay、toast、モバイル表示を定義する CSS。
- メソッド一覧: なし。
