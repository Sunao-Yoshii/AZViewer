- メソッド: なし。

## `backend`

### `backend/main.py`
- 概要: pywebview アプリケーションのエントリポイント。Vue の表示 URL 解決、ドラッグ＆ドロップイベント登録、ウィンドウ生成、API ライフサイクル管理を行う。
- メソッド:
  - `resolve_entry_url()`: Vite 開発サーバー URL または `frontend/dist/index.html` の URL を返す。
  - `create_drop_items(event)`: pywebview の drop イベントから登録対象ファイル/フォルダ情報を作る。
  - `register_handlers(window, api)`: DOM の drop イベントを pywebview ウィンドウに登録する。
  - `main()`: アプリケーションを起動し、終了コードを返す。

### `backend/api/api_response.py`
- 概要: Vue 側へ返す API レスポンス形式を統一する dataclass。
- クラス:
  - `ApiResponse`
- メソッド:
  - `to_dict()`: `success`、`message`、`data` を持つ辞書へ変換する。

### `backend/api/app_api.py`
- 概要: pywebview に公開するアプリケーション API の集約クラス。ライフサイクル、画像登録、画像検索/更新、OS 操作 API へ処理を委譲する。
- クラス:
  - `AppApi`
- メソッド:
  - `__init__()`: API クラスと共有サービスを組み立てる。
  - `close()`: アプリ終了時に DB 接続などを閉じる。
  - `initialize()`: 互換用の初期化 API。
  - `initialize_app()`: Vue 起動用の基本情報を返す。
  - `bootstrap_app()`: 起動時整合性確認を含む初期化結果を返す。
  - `get_app_info()`: アプリ名やバージョンを返す。
  - `health_check()`: Python API の疎通確認結果を返す。
  - `import_selected_items(payload)`: 選択/ドロップされたファイルを登録する。
  - `select_files_dialog()`: ファイル選択ダイアログを開く。
  - `select_folder_dialog()`: フォルダ選択ダイアログを開く。
  - `search_image_files(payload)`: 画像一覧を検索する。
  - `update_image_file_detail(payload)`: 画像詳細項目を更新する。
  - `delete_image_file(payload)`: 画像登録情報を削除する。
  - `fetchLocalImage(payload)`: ローカル画像本体を data URL として返す。
  - `fetchLocalImageThumb(payload)`: サムネイルを data URL として返す。
  - `open_containing_folder(payload)`: 対象ファイルの存在するフォルダを開く。
  - `_get_active_window()`: 現在の pywebview ウィンドウを返す。

### `backend/api/app_lifecycle_api.py`
- 概要: アプリ起動・終了時の処理を担当する API。DB 初期化、起動時クリーンアップ、不足サムネイル生成、初期検索を行う。
- クラス:
  - `AppLifeCycleApi`
- メソッド:
  - `__init__(...)`: 起動処理に必要な API/サービスを保持する。
  - `initialize()`: 互換用に `bootstrap_app()` を呼ぶ。
  - `bootstrap_app()`: 初期化、整合性確認、初期検索結果を返す。
  - `close()`: DB 接続を閉じる。
  - `_run_startup_cleanup()`: 存在しない登録ファイルを削除し、通知情報を作る。
  - `_get_cleanup_service()`: 起動時クリーンアップサービスを取得する。
  - `_ensure_thumbnail_cache()`: 既存レコードの不足サムネイルを補完する。
  - `_get_cleanup_repository()`: 起動時処理用リポジトリを取得する。

### `backend/api/database_lifecycle_manager.py`
- 概要: アプリケーションで共有する SQLite 接続のライフサイクル管理に限定したクラス。
- クラス:
  - `DatabaseLifecycleManager`
- メソッド:
  - `__init__()`: `ConnectionManager` を準備する。
  - `initialize()`: 共有 SQLite 接続を初期化する。
  - `close()`: 共有 SQLite 接続を閉じる。
  - `get_connection()`: 初期化済みの共有接続を返す。

### `backend/api/default_template_api.py`
- 概要: アプリケーション基本情報とヘルスチェックを返す API。
- クラス:
  - `DefaultTemplateApi`
- メソッド:
  - `initialize_app()`: Vue 起動時に必要な初期化情報を返す。
  - `get_app_info()`: アプリ名、バージョン、説明を返す。
  - `health_check()`: Python API の疎通確認レスポンスを返す。

### `backend/api/image_catalog_api.py`
- 概要: 画像一覧の検索、詳細更新、削除、画像本体/サムネイル取得を提供する API。
- クラス:
  - `ImageCatalogApi`
- メソッド:
  - `__init__(...)`: DB ライフサイクル管理とサムネイルサービスを保持する。
  - `search_image_files(payload)`: 条件、ページング、ソートに基づき画像一覧を検索する。
  - `update_image_file_detail(payload)`: rating、チェック、お気に入り、コメントを更新する。
  - `delete_image_file(payload)`: レコードと対応サムネイルを削除する。
  - `fetchLocalImage(payload)`: ローカル画像本体を data URL として返す。
  - `fetchLocalImageThumb(payload)`: サムネイル PNG を data URL として返す。
  - `_get_repository()`: 一覧系リポジトリを遅延初期化して返す。
  - `_normalize_nullable_string(value)`: 空文字列を `None` へ正規化する。
  - `_normalize_detail_payload(data)`: 詳細更新 payload をリポジトリ用に正規化する。
  - `_normalize_detail_comment(value)`: コメントの空文字を `None` にし、255 文字以内を検証する。

### `backend/api/image_register_api.py`
- 概要: 画像登録に関する pywebview API。ファイル/フォルダ選択ダイアログと登録処理を提供する。
- クラス:

- メソッド:
  - `__init__(...)`: DB、ダイアログ、サムネイルサービスを保持する。
  - `_get_import_service()`: 登録サービスを遅延初期化して返す。
  - `import_selected_items(payload)`: 選択またはドロップされた items を登録する。
  - `select_files_dialog()`: ネイティブのファイル選択ダイアログを開く。
  - `select_folder_dialog()`: ネイティブのフォルダ選択ダイアログを開く。

### `backend/api/os_operation_api.py`
- 概要: OS 依存の操作を担当する API。現在は Windows Explorer で対象ファイルの親フォルダを開く。

- メソッド:

### `backend/db/connection_manager.py`
- 概要: SQLite データベースファイルと接続を管理する低レベルクラス。
- クラス:
  - `ConnectionManager`
- メソッド:
  - `__init__(db_path=None)`: DB パスと接続状態を初期化する。
  - `db_path`: 現在の DB ファイルパスを返す property。
  - `initialize()`: SQLite 接続を作成し、row factory と外部キー設定を適用する。
  - `get_connection()`: 初期化済み接続を返す。
  - `close()`: SQLite 接続を閉じる。

### `backend/models/image_file.py`
- 概要: 画像ファイル登録、一覧表示、検索結果、インポート結果の DTO/dataclass を定義する。
- クラス:
  - `ImageFileRecord`: DB 登録用の画像ファイル情報。
  - `ImageFileListItem`: 一覧表示用の画像ファイル情報。
  - `SearchImageFilesResult`: ページング付き検索結果。
  - `ImportResult`: 画像登録処理の結果。
- メソッド:
  - `ImageFileListItem.to_dict()`: API 返却用辞書へ変換する。
  - `SearchImageFilesResult.to_dict()`: items とページング情報を API 返却用辞書へ変換する。
  - `ImportResult.to_api_data()`: 登録成功/失敗に応じた API data を作る。

### `backend/repositories/image_file_repository.py`
- 概要: SQLite の `image_file_data` テーブルへの永続化、検索、更新、削除を担当するリポジトリ。
- クラス:
  - `ImageFileRepository`
- メソッド:
  - `__init__(connection)`: SQLite 接続を保持する。
  - `create_table()`: `image_file_data` テーブルを作成する。
  - `find_all_paths()`: 登録済みパス一覧を返す。
  - `find_all_items()`: 登録済み画像を全件取得する。
  - `delete_by_paths(paths)`: 指定パスのレコードを削除し、削除 ID を返す。
  - `insert_many(records)`: 複数画像レコードを一括登録する。
  - `exists_by_path(path)`: 指定パスが登録済みか判定する。
  - `find_by_paths(paths)`: 指定パス群の一覧項目を取得する。
  - `find_ids_by_paths(paths)`: 指定パス群の ID 一覧を取得する。
  - `search(...)`: 条件、ページング、ソートに基づき検索する。
  - `delete_by_id(record_id)`: ID 指定で削除する。
  - `update_detail(...)`: rating、チェック、お気に入り、コメントを更新する。
  - `find_by_id(record_id)`: ID 指定で 1 件取得する。
  - `_row_to_item(row)`: SQLite 行を `ImageFileListItem` へ変換する。
  - `_build_order_by(sort)`: 許可済み sort 値から ORDER BY 句を作る。
  - `_normalize_true_condition(value)`: 検索条件の true 判定を行う。
  - `_validate_detail_values(...)`: 詳細更新値を検証する。

### `backend/services/file_scan_service.py`
- 概要: 登録対象画像ファイルの抽出と DB 登録用レコード生成を担当するサービス。
- クラス:
  - `FileScanService`
- メソッド:
  - `is_supported_image(path)`: 対応拡張子の画像ファイルか判定する。
  - `collect_image_paths(items)`: ファイル/フォルダ指定から登録対象画像パスを収集する。
  - `build_record(path)`: 画像パスから `ImageFileRecord` を作る。
  - `_collect_direct_children(directory)`: フォルダ直下の対応画像のみ収集する。
  - `_dedupe_paths(paths)`: 順序を維持して重複パスを除去する。

### `backend/services/image_file_import_service.py`
- 概要: 選択/ドロップされたファイル・フォルダから未登録画像を DB に登録し、サムネイルを生成するサービス。
- クラス:
  - `ImageFileImportService`
- メソッド:
  - `__init__(...)`: DB 接続、リポジトリ、スキャン、サムネイルサービスを保持する。
  - `import_items(payload)`: payload を解析し、未登録画像の登録とサムネイル生成を行う。

### `backend/services/startup_cleanup_service.py`
- 概要: 起動時に実ファイルが存在しない登録情報を削除し、関連サムネイルも削除するサービス。
- クラス:
  - `StartupCleanupService`
- メソッド:
  - `__init__(...)`: DB 接続、リポジトリ、サムネイルサービスを保持する。
  - `cleanup_missing_files()`: 存在しないファイルの登録情報を削除し、削除件数を返す。

### `backend/services/thumbnail_cache_service.py`
- 概要: 画像サムネイル PNG の生成、取得、削除を担当するサービス。サムネイル長辺は 300px。
- クラス:
  - `ThumbnailCacheService`
- メソッド:
  - `__init__(cache_dir=None)`: サムネイル保存先を初期化する。
  - `ensure_thumbnails(items, missing_only=True)`: 一覧項目に対して必要なサムネイルを生成する。
  - `create_thumbnail(record_id, source_path)`: 指定画像から PNG サムネイルを生成する。
  - `delete_thumbnails(record_ids)`: 複数 ID のサムネイルを削除する。
  - `delete_thumbnail(record_id)`: 指定 ID のサムネイルを削除する。
  - `get_thumbnail_bytes(record_id)`: サムネイルファイルの bytes を返す。
  - `get_thumb_path(record_id)`: サムネイル保存パスを返す。

- 概要: アプリ全体の最上位 Vue コンポーネント。検索状態、一覧データ、詳細モーダル、トースト、ローディングオーバーレイ、backend API 呼び出しを統括する。

  - `handlePageChange(page)`: ページ変更を処理する。
  - `handlePageSizeChange(pageSize)`: 表示件数変更を処理する。
  - `handleSortChange(sort)`: 並び順変更を処理する。
  - `handleOpenDetail(item)`: 詳細モーダルを開き、画像本体を読み込む。
  - `handleCloseDetail()`: 詳細モーダルを閉じる。
  - `handleOpenContainingFolder(path)`: 対象ファイルのフォルダを Explorer で開く API を呼ぶ。
  - `handleDelete(id)`: 削除確認後、画像登録情報を削除する。
  - `handleSaveDetail(payload)`: タイル内編集の保存 API を呼び、成功時に再検索する。
  - `handleImportedEvent()`: pywebview drop 登録完了イベントを処理する。
