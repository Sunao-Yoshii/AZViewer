
# 実装仕様書

## 画像タグ管理機能追加

## 1. 目的

画像に対してタグ情報を付与・表示・編集できるようにする。
タグは将来のタグ検索を前提とし、`tag` マスタテーブルと `tag_image_link` リンクテーブルで管理する。

既存の画像一覧・更新系は `ImageCatalogApi`、`ImageFileRepository`、`ImageFileListItem` を中心に構成されているため、この流れにタグ情報を追加する。

## 3. DB仕様

### 3.1 tag

```sql
CREATE TABLE IF NOT EXISTS tag (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,
    CHECK (length(name) <= 128)
);
```

### 3.2 tag_image_link

```sql
CREATE TABLE IF NOT EXISTS tag_image_link (
    image_file_id INTEGER NOT NULL,
    tag_id INTEGER NOT NULL,
    PRIMARY KEY (image_file_id, tag_id),
    FOREIGN KEY (image_file_id)
        REFERENCES image_file_data(id)
        ON DELETE CASCADE,
    FOREIGN KEY (tag_id)
        REFERENCES tag(id)
        ON DELETE CASCADE
);
```

### 3.3 削除時

`image_file_data` 削除時、`tag_image_link` は外部キー制約で削除される。

`tag` は削除しない。
未使用タグの掃除は今回対象外。

## 4. タグ正規化仕様

### 4.1 入力分割

`,` のみ区切り文字とする。

```text
"girl, blue eyes, smile"
→ ["girl", "blue eyes", "smile"]
```

全角カンマ `，`、読点 `、` は区切り文字扱いしない。

### 4.2 正規化順序

```text
1. "," で分割
2. trim
3. 禁止文字 []{}() を除去
4. 末尾の :数値 を除去
5. trim 再実行
6. 小文字化
7. 空文字除去
8. 128文字超過はエラー
9. 正規化後重複は黙って統合
```

### 4.3 禁止文字

以下を除去する。

```regex
[\[\]\{\}\(\)]
```

例。

```text
"(masterpiece)" → "masterpiece"
"[blue eyes]"  → "blue eyes"
```

### 4.4 ウェイト除去

末尾が `:数値` の場合のみ除去する。

```regex
:\d+(?:\.\d+)?$
```

例。

```text
girl:1.2       → girl
blue eyes:0.8  → blue eyes
artist:name    → artist:name
tag:abc        → tag:abc
```

```

```

### 6.3 replace_tags 処理

```text
1. tag_image_link から対象 image_file_id のリンクを削除
2. tags を順に処理
3. tag に存在しなければ INSERT
4. tag_id を取得
5. tag_image_link に INSERT
```

`tag_image_link` は `PRIMARY KEY (image_file_id, tag_id)` のため、重複リンクはDB上も防止する。

### 6.4 search 結果へのタグ付与

`search(...)` で画像一覧を取得後、取得した `id` 一覧からタグをまとめて取得する。

```text
1. image_file_data を検索
2. items を生成
3. item ids を抽出
4. find_tags_by_image_ids(ids)
5. 各 ImageFileListItem.tags に設定
```

N+1クエリは禁止。ここで1件ずつタグ取得すると、検索が「小刻みに足を引っ張る妖怪」になる。

## 7. API仕様

### 7.1 update_image_file_detail

既存 payload に `tags` を追加する。
既存の更新経路は `edit.vue → tile.vue → Content.vue → App.vue → update_image_file_detail` の流れを維持する。

```json
{
  "id": 1,
  "rating": "General",
  "is_checked": 1,
  "is_favorite": 0,
  "comment": "memo",
  "tags": ["masterpiece", "blue eyes", "smile"]
}
```

### 7.2 API処理

```text
1. payload 検証
2. comment 正規化
3. tags 正規化
4. image_file_data 更新
5. tag / tag_image_link 更新
6. 成功レスポンス返却
```

### 7.3 トランザクション

`image_file_data` 更新とタグリンク更新は同一トランザクションで扱う。

```text
BEGIN
  update_detail
  replace_tags
COMMIT
```

失敗時は rollback。

### 8.5 addTagsFromInput

```js
function normalizeTagText(value) {
  return value
    .trim()
    .replace(/[\[\]\{\}\(\)]/g, '')
    .replace(/:\d+(?:\.\d+)?$/, '')
    .trim()
    .toLowerCase()
}

function addTagsFromInput() {
  const normalizedTags = form.tagInput
    .split(',')
    .map(normalizeTagText)
    .filter(Boolean)

  const tooLong = normalizedTags.find((tag) => tag.length > 128)
  if (tooLong) {
    // トーストまたはインラインエラー
    return
  }

  form.tags = [...new Set([...form.tags, ...normalizedTags])]
  form.tagInput = ''
}
```

### 8.6 removeTag

```js
function removeTag(tag) {
  form.tags = form.tags.filter((value) => value !== tag)
}
```

### 8.7 isDirty

タグ比較を追加する。

```js
function normalizeTagsForCompare(tags) {
  return [...(tags ?? [])].sort()
}

const isDirty = computed(() => (
  form.rating !== props.item.rating ||
  form.is_checked !== props.item.is_checked ||
  form.is_favorite !== props.item.is_favorite ||
  form.comment !== (props.item.comment ?? '') ||
  JSON.stringify(normalizeTagsForCompare(form.tags)) !==
    JSON.stringify(normalizeTagsForCompare(props.item.tags))
))
```

## 9. 保存フロー

既存フローを維持する。

```text
edit.vue
  ↓ save emit
tile.vue
  ↓ save-detail emit
Content.vue
  ↓ save-detail emit
App.vue
  ↓ useImageMutations.handleSaveDetail
  ↓ updateImageFileDetail
  ↓ 成功時 再検索
```

更新・削除は `useImageMutations.js` に集約されているため、コンポーネントから直接 API を呼ばない方針を維持する。

失敗時も既存通り。

```text
edit表示維持
入力内容保持
トースト通知
再検索しない
```

この失敗時挙動は既存仕様とも一致する。

## 11. 受け入れ条件

### DB

* `tag` テーブルが作成されること
* `tag_image_link` テーブルが作成されること
* 同一タグ名が `tag` に重複登録されないこと
* 同一画像に同一タグリンクが重複登録されないこと
* 画像削除時に `tag_image_link` が削除されること

### 正規化

* タグは小文字で保存されること
* `[]{}()` が除去されること
* 末尾の `:1.2` 形式が除去されること
* 全角カンマ、読点では分割されないこと
* 正規化後の重複タグは黙って1つに統合されること
* 128文字超過タグは保存できないこと

### 表示

* `display.vue` にタグが灰色 badge で表示されること
* タグ0件時に不要な空欄が出ないこと

### 編集

* `edit.vue` でタグを入力できること
* Enterでタグ追加できること
* `,` 区切りで複数タグ追加できること
* xボタンでタグ削除できること
* 「保存して戻る」時のみDB反映されること
* 保存失敗時に入力中タグが消えないこと

### 一覧取得

* `search_image_files` の各 item に `tags` が含まれること
* タグ取得がN+1クエリにならないこと
