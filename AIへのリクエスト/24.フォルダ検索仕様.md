# 追加仕様書

## フォルダ検索機能追加

## 1. 目的

画像検索フォームに `image_file_data.folder` を対象としたフォルダ検索条件を追加する。

既存のタグ検索ダイアログと類似したインターフェースでフォルダを選択できるようにする。ただし、タグ検索とは異なり、フォルダは **1件のみ選択可能** とする。

---

## 2. 変更概要

以下の機能を追加する。

```text
- search_form.vue にフォルダ検索UIを追加
- FolderSearchModal.vue を新規追加
- useImageSearch.js の filters / 検索 payload に folder を追加
- backendApi.js に fetchFoldersForSearch を追加
- AppApi に fetch_folders_for_search を追加
- ImageCatalogApi に fetch_folders_for_search を追加
- ImageFileRepository.search に folder 条件を追加
- ImageFileRepository にフォルダ候補取得メソッドを追加
```

## 4.2 FolderSearchModal.vue

### 4.2.1 役割

`FolderSearchModal.vue` は、検索条件に使用するフォルダを選択するためのモーダルダイアログとする。

このコンポーネントは、画像検索そのものは実行しない。
選択したフォルダを `search_form.vue` に適用するだけとする。

### 4.2.2 表示内容

```text
- モーダルタイトル
- 選択中フォルダ badge
- フィルタ用テキストボックス
- 検索ボタン
- 表示件数 / 現在の条件に一致するフォルダ総数
- フォルダ一覧
- OK ボタン
- キャンセルボタン
```

### 4.2.3 モーダルタイトル

```text
フォルダ検索
```

### 4.2.4 選択中フォルダ

ダイアログ内で、現在の仮選択フォルダを badge 表示する。

```text
選択中:
[set01 ×]
```

`×` をクリックすると、仮選択フォルダを解除する。

### 4.2.5 フィルタ欄

フィルタ用テキストボックスの右に検索ボタンを配置する。

```text
[ フォルダ名で検索 ][ 検索 ]
```

`input` イベントでは検索しない。
検索ボタン押下時にフォルダ候補取得 API を実行する。

Enter キー押下時も、検索ボタン押下と同じ処理を実行してよい。

```html
<div class="input-group input-group-sm">
  <input
    v-model="filterText"
    type="text"
    class="form-control"
    placeholder="フォルダ名で検索"
    @keydown.enter.prevent="handleSearch"
  />

  <button
    type="button"
    class="btn btn-outline-secondary"
    @click="handleSearch"
  >
    検索
  </button>
</div>
```

### 4.2.6 表示件数

フォルダ一覧の上部に、現在表示中の件数と、現在の条件に一致するフォルダ総数を表示する。

表示形式。

```text
表示件数: {表示中フォルダ数} / {条件一致総数}
```

例。

```text
表示件数: 12 / 42
```

最大表示件数を超える場合。

```text
表示件数: 256 / 1840
```

この場合、一覧に表示されるのは最大256件までとする。

### 4.2.7 フォルダ一覧

フォルダ一覧は `image_file_data.folder` の重複なし一覧から取得する。

取得条件。

```text
- フィルタ未指定時は全フォルダ候補
- フィルタ指定時は folder 部分一致
- 最大256件
- 登録順相当で表示
```

登録順相当とは、各 folder に属する画像レコードの最小 `id` 昇順とする。

```sql
ORDER BY MIN(id) ASC
```

フォルダ一覧は badge またはボタン形式で表示する。

```html
<button
  type="button"
  class="badge text-bg-light border"
>
  set01
</button>
```

### 4.2.8 フォルダ選択

フォルダ一覧の項目をクリックすると、仮選択フォルダに設定する。

フォルダは1件のみ選択可能とする。

別のフォルダをクリックした場合、既存の仮選択フォルダを置き換える。

```text
選択中: set01
↓
set02 をクリック
↓
選択中: set02
```

### 4.2.9 OKボタン

OKボタン押下時、仮選択フォルダを `search_form.vue` の `filters.folder` に反映し、ダイアログを閉じる。

OK押下だけでは画像検索を実行しない。

### 4.2.10 キャンセル

キャンセルボタン、または右上 `×` で閉じた場合、`search_form.vue` の `filters.folder` は変更しない。

```python
def _normalize_search_folder(self, value) -> str | None:
    folder = str(value or "").strip()
    return folder or None
```

### 7.2.3 fetch_folders_for_search 追加

```python
def fetch_folders_for_search(self, payload):
    try:
        data = payload or {}
        keyword = str(data.get("keyword") or "").strip()

        try:
            limit = int(data.get("limit") or 256)
        except (TypeError, ValueError):
            limit = 256

        limit = max(1, min(limit, 256))

        repository = self._get_repository()

        folders = repository.find_folders_for_search(
            keyword=keyword,
            limit=limit,
        )
        total_count = repository.count_folders_for_search(
            keyword=keyword,
        )

        return ApiResponse(
            success=True,
            message="",
            data={
                "folders": [folder.to_dict() for folder in folders],
                "total_count": total_count,
                "limit": limit,
            },
        ).to_dict()

    except Exception:
        return ApiResponse(
            success=False,
            message="フォルダ一覧を取得できませんでした。",
            data={
                "folders": [],
                "total_count": 0,
                "limit": 256,
            },
        ).to_dict()
```

### 7.5.1 find_folders_for_search

```python
def find_folders_for_search(
    self,
    keyword: str | None = None,
    limit: int = 256,

    ...
```

SQL。

```sql
SELECT
    folder AS name,
    COUNT(*) AS image_count,
    MIN(id) AS first_id
FROM image_file_data
WHERE
    (:keyword = '' OR folder LIKE :keyword_like)
GROUP BY folder
ORDER BY first_id ASC
LIMIT :limit
```

### 7.5.2 count_folders_for_search

```python
def count_folders_for_search(
    self,
    keyword: str | None = None,

    ...
```

SQL。

```sql
SELECT
    COUNT(*)
FROM (
    SELECT
        folder
    FROM image_file_data
    WHERE
        (:keyword = '' OR folder LIKE :keyword_like)
    GROUP BY folder
) folders
```

### 7.5.3 LIKE パラメータ

## 8. SQL仕様

## 8.1 フォルダ未指定時

フォルダ条件は付与しない。

```sql
WHERE
    ...
```

## 8.2 フォルダ指定時

```sql
WHERE
    ...
    AND i.folder = :folder
```

## 8.3 タグ条件との併用

```sql
WHERE
    ...
    AND i.folder = :folder
    AND i.id IN (
        SELECT
            til.image_file_id
        FROM tag_image_link til
        INNER JOIN tag t
            ON t.id = til.tag_id
        WHERE t.name IN (:tag_0, :tag_1)
        GROUP BY til.image_file_id
        HAVING COUNT(DISTINCT t.name) = :tag_count
    )
```

## 12. 注意点

フォルダ検索は `folder` 名の完全一致で行う。
候補一覧のフィルタは部分一致だが、画像検索時は完全一致である。

```text
候補検索:
  folder LIKE '%set%'

画像検索:
  image_file_data.folder = 'set01'
```

この違いは意図的なものとする。候補選択は探すための検索、画像検索は選んだフォルダで確定的に絞るための検索である。
ここを両方部分一致にすると、「set」で検索したつもりが `set01`, `set02`, `preset` まで混ざる可能性があり、フォルダ検索というよりフォルダ占いになります。
