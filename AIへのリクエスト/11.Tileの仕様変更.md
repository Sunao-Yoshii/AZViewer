# 実装仕様書

## 画像タイル表示・詳細モーダル機能拡張

---

## 1. 目的

画像一覧画面におけるタイル表示の操作性および情報表示を改善し、
画像詳細の参照・編集・削除操作を一貫したUIで提供する。

---

## 2. 変更概要

以下の機能を追加・変更する。

* タイルUIの改善（ID削除・バッジ化・削除ボタン追加）
* 詳細表示用モーダルの新規追加
* 画像本体取得APIの追加（path指定）
* レコード削除APIの追加
* 詳細更新APIの追加（複数項目更新）
* UIイベントの親コンポーネント集約

---

## 3. タイルコンポーネント仕様 (`tile.vue`)

### 3.1 表示項目

| 項目          | 表示             |
| ----------- | -------------- |
| サムネイル       | 表示             |
| filename    | 1行省略 + tooltip |
| folder      | 1行省略 + tooltip |
| rating      | Badge          |
| is_checked  | Badge          |
| is_favorite | Badge          |
| comment     | 表示             |

※ `id` 表示は削除

---

### 3.2 フラグ表示仕様

| 項目          | true                         | false                |
| ----------- | ---------------------------- | -------------------- |
| is_checked  | `badge bg-primary`           | `badge bg-secondary` |
| is_favorite | `badge bg-warning text-dark` | `badge bg-secondary` |

表示文言:

* is_checked → **チェック**
* is_favorite → **お気に入り**

---

### 3.3 削除ボタン

#### 配置

* タイル右上
* Bootstrap の `btn-close` 使用

#### 挙動

クリック時:

```javascript
emit('request-delete', item.id)
```

---

### 3.4 画像クリック

クリック時:

```javascript
emit('open-detail', item)
```

---

### 3.5 責務

含む:

* 表示
* 軽量イベント発火

含まない:

* API呼び出し
* 状態管理
* トースト
* 再検索

---

## 4. 詳細モーダル仕様 (`ImageDetailModal.vue`)

---

### 4.1 モーダル基本

| 項目   | 内容                           |
| ---- | ---------------------------- |
| サイズ  | `modal-xl`                   |
| 表示位置 | 中央 (`modal-dialog-centered`) |
| 閉じる  | 右上 × のみ                      |
| フッター | Saveボタンのみ                    |

---

### 4.2 タイトル

```text
{filename}
```

---

### 4.3 レイアウト

```text
┌──────────────────────────────┬──────────────┐
│                              │              │
│      画像表示領域 (80%)       │  入力フォーム │
│                              │    (20%)     │
│                              │              │
└──────────────────────────────┴──────────────┘
```

---

### 4.4 左側（画像表示）

#### 画像取得

API:

```json
fetchLocalImage({
  "path": item.path
})
```

---

#### 表示仕様

* 縮小表示（デフォルト）
* 画面より大きい場合 → 縮小
* 小さい場合 → 中央配置

CSS例:

```css
img {
  max-width: 100%;
  max-height: calc(100vh - 200px);
  object-fit: contain;
}
```

---

#### クリック動作

| 状態   | 動作   |
| ---- | ---- |
| 縮小状態 | 原寸表示 |
| 原寸表示 | 縮小表示 |

---

### 4.5 右側（入力フォーム）

---

#### 上部表示

* path を全文表示
* 改行許可

```html
<div class="small text-muted detail-path">
  {{ item.path }}
</div>
```

```css
.detail-path {
  white-space: pre-wrap;
  overflow-wrap: anywhere;
}
```

---

#### 入力項目

---

##### ① チェック

* トグルスイッチ
* 表示名: **チェック**

---

##### ② お気に入り

* トグルスイッチ

---

##### ③ rating

Radio group:

* General
* R-15
* R-18
* R-18G

---

##### ④ comment

* textarea
* 高さ: 5行
* 改行可

---

### 4.6 Saveボタン

クリック時:

```javascript
confirm("入力内容を保存しますか？")
```

---

#### OK時

```javascript
emit('save', {
  id,
  rating,
  is_checked,
  is_favorite,
  comment
})
```

---

#### 保存成功時

* モーダルを閉じる
* 親へ通知

## 5. API仕様

---

### 5.1 画像本体取得

#### `fetchLocalImage`

```json
{
  "path": "C:/images/sample.png"
}
```

#### 処理

* ファイル読み込み
* バイナリ返却

---

### 5.2 レコード削除

#### `delete_image_file`

```json
{
  "id": 1
}
```

#### 処理

* DB削除
* `data/thumbs/{id}.png` 削除

---

### 5.3 詳細更新

#### `update_image_file_detail`

```json
{
  "id": 1,
  "rating": "General",
  "is_checked": 1,
  "is_favorite": 0,
  "comment": "memo"
}
```

#### 処理

* 一括更新
* バリデーション実施

## 8. 状態遷移

---

### 削除

```text
tile → 削除クリック
→ confirm
→ delete API
→ 再検索
```

---

### 詳細表示

```text
tile → 画像クリック
→ モーダル表示
→ 編集
→ Save
→ confirm
→ 更新API
→ 再検索
→ モーダル閉じる
```

---

## 9. 注意事項

* tile にロジックを持たせない
* API呼び出しは親へ集約
* DBアクセスを最小化（path直取得）
* UI操作は一方向データフロー維持

---

## 10. 補足

* タイルと詳細で編集機能を二重化しない
* 状態更新は必ず API 経由
* UIの責務肥大化を防ぐ
