# 実装仕様書

## LoRA caption 向けタグ読み込み・一括タグ追加・タグ操作メニュー再編

## 1. 目的

LoRA 作成向けに、画像と同名の caption `.txt` ファイルからタグを読み込み、AZViewer 上の画像タグへ追加登録できるようにする。

あわせて、チェックした画像へ同一タグを一括追加する機能を追加する。

さらに、既存のタグ出力機能について、caption 内のタグ出力順を **選択画像内での出現頻度順** に変更する。

---

## 2. 変更概要

以下を追加・変更する。

```text
- AppHeader.vue に「タグ操作」ドロップダウンを追加
  - プロンプトの読み取り
  - キャプションタグ読み込み
  - 一括タグ追加
  - 重複タグ構成の検索

- キャプションタグ読み込み機能を追加
  - 選択画像と同じディレクトリにある同名 .txt を読み込む
  - caption 内タグを既存タグへ追加登録する
  - 置き換えは行わない

- 一括タグ追加機能を追加
  - ダイアログでカンマ区切りタグを入力する
  - 選択画像すべてへ同一タグを追加登録する
  - 置き換えは行わない

- タグ出力機能を変更
  - 出力順を tag.id 昇順から、選択画像内での出現頻度順へ変更する
  - 出力処理中は LoadingOverlay で画面ロックする
```

## 4. 基本方針

## 4.1 選択状態は既存仕様を流用する

画像選択状態は DB に保存しない。

既存の一括操作と同様に、`App.vue` の一時状態である `selectedImageIds` / `selectedImageIdSet` を利用する。

```text
- DB カラム追加なし
- image_file_data に選択状態を持たせない
- チェックされた画像 ID を API payload として送る
```

## 4.2 タグ追加は既存タグを保持する

caption 読み込み・一括タグ追加のどちらも、既存タグを置き換えない。

```text
既存タグ + 追加タグ
```

禁止。

```text
既存タグを削除して caption タグだけに置き換える
既存タグを削除して一括入力タグだけに置き換える
```

ここは明記必須です。
実装者が `replace_tags` をうっかり呼ぶと、caption 読み込み機能が「追加」ではなく「タグ消しゴム」になります。消しゴムにしては殺意が高い。

## 4.3 タグ正規化は既存ルールを使う

caption 読み込み・一括タグ追加の入力文字列は、既存の `TagNormalizeService` 相当の正規化ルールを通す。

想定ルール。

```text
- ASCII カンマ "," で分割
- 前後空白を trim
- 空タグ除外
- 禁止文字除去
- 末尾の :数値 を除去
- 小文字化
- 重複除去
- タグ単体の最大長制限を適用
```

## 4.4 tag_hash を必ず更新する

caption 読み込み・一括タグ追加は、画像のタグ構成を変更する。

そのため、タグリンク更新と `tag_hash` 更新を同一トランザクションで扱う。

既存仕様でも、タグ編集保存時は `replace_tags` と `replace_tag_hash` を同一フローで実行し、プロンプト一括タグ登録時にも `replace_tag_hash` を更新する流れになっています。

追加対象。

```text
- キャプションタグ読み込み成功画像
- 一括タグ追加成功画像
```

## 4.5 処理単位

caption 読み込み・一括タグ追加は、どちらも **1画像単位で処理継続** とする。

```text
1画像で失敗しても、他画像の処理は継続する
```

プロンプト一括読み取りでも、1件ずつ処理して失敗詳細を集計する設計になっているため、それに合わせる。

# 6. キャプションタグ読み込み仕様

## 6.1 概要

チェックした画像に対して、画像ファイルと同じディレクトリにある同名 `.txt` を読み込み、画像タグとして追加登録する。

例。

```text
画像:
C:/images/sample.png

読み込み対象:
C:/images/sample.txt
```

## 6.2 対象

```text
selectedImageIds に含まれる画像
```

## 6.3 ファイル探索

Python 側で対象画像を ID から再取得する。

```text
フロント:
ids のみ送信

バックエンド:
find_by_ids(ids) で path を取得
Path(item.path).with_suffix(".txt") で caption パスを生成
```

フロントから path を送らない。
削除・移動系と同じく、対象は DB 登録済み画像に限定するべきです。フロントから path を受け始めると、入力欄が急にファイル破壊光線になります。

## 6.4 読み込み文字コード

```text
UTF-8
```

BOM 付き UTF-8 は許容してよい。

推奨実装。

```python
caption_path.read_text(encoding="utf-8-sig")
```

Shift_JIS 自動判定は初回仕様に含めない。
必要になったら別仕様でよいです。文字コード自動判定は、見た目より沼です。水たまりかと思ったら底なしタイプ。

## 6.5 タグ追加ルール

```text
1. caption txt の本文を読み込む
2. TagNormalizeService でタグ化する
3. 既存タグを取得する
4. 既存タグ + caption タグを統合する
5. 重複除去する
6. tag_image_link を更新する
7. tag_hash を更新する
```

## 6.6 スキップ条件

以下は失敗ではなくスキップ。

```text
- 同名 .txt が存在しない
- .txt が空
- 正規化後タグが 0 件
- 追加対象タグがすべて既存タグと重複し、実質的な変更がない
```

## 6.7 失敗条件

```text
- caption ファイル読み込み失敗
- 文字コードエラー
- DB 更新失敗
- tag_hash 更新失敗
```

失敗詳細は最大 20 件まで返す。既存の一括処理系 DTO でも失敗詳細は最大 20 件の方針が使われています。

## 6.8 確認文言

```text
選択した {count} 件の画像について、画像と同名の caption .txt を読み込み、タグとして追加登録します。

既存タグは保持されます。
同名 .txt が存在しない画像、タグが取得できない画像はスキップされます。

実行しますか？
```

## 6.9 LoadingOverlay

```text
タイトル: キャプションタグを読み込み中
メッセージ: 選択画像と同名の caption ファイルからタグを追加しています...
```

## 6.10 完了 toast

成功・スキップのみ。

```text
キャプションタグ読み込みが完了しました。対象: X 件 / 更新: Y 件 / スキップ: Z 件 / 失敗: 0 件
```

失敗あり。

```text
キャプションタグ読み込みが完了しましたが、一部失敗しました。対象: X 件 / 更新: Y 件 / スキップ: Z 件 / 失敗: W 件
```

## 6.11 完了後処理

API 成功時。

```text
- clearSelection()
- 現在検索条件で再検索
```

# 8. タグ出力仕様変更

## 8.1 変更対象

既存の「タグ出力」。

既存仕様では、選択画像のタグを画像と同名 `.txt` に出力し、既存 `.txt` は上書き、タグ 0 件画像はスキップ、DB 更新は行わない方針です。

## 8.2 出力順序

変更前。

```text
tag.id 昇順
```

変更後。

```text
選択画像内でのタグ出現頻度順
```

## 8.3 出現頻度の定義

対象範囲。

```text
タグ出力対象として選択された画像群
```

頻度の数え方。

```text
そのタグを含む画像数
```

1画像内で同一タグが重複していても、1件として数える。

ただし、既存タグ管理上は画像内重複タグは基本的に排除される前提。

## 8.4 並び順

各 caption ファイル内のタグ順序は以下。

```text
1. 選択画像内での出現画像数 DESC
2. 同頻度の場合は tag.name ASC
```

例。

```text
選択画像:
A: 1girl, blue eyes, smile
B: 1girl, smile
C: 1girl, black hair

出現頻度:
1girl = 3
smile = 2
black hair = 1
blue eyes = 1

A の出力:
1girl, smile, blue eyes
```

## 8.5 DB 取得

出力時点の DB 状態を正とするため、既存どおりバックエンド側で対象画像 ID からタグを再取得する。既存 Repository には画像 ID ごとのタグ一覧取得である `find_tags_by_image_ids(image_ids)` が定義されています。

ただし、今回は頻度計算に `tag.name` が必要なため、内部的には以下のような構造が望ましい。

```python
{
    image_id: [
        {"id": 1, "name": "1girl"},
        {"id": 2, "name": "smile"},
    ]
}
```

現行の `find_tags_by_image_ids` が `dict[int, list[str]]` を返す場合は、以下のどちらかで対応する。

```text
A. find_tags_by_image_ids の戻り値を既存互換のまま使い、name ベースで頻度計算する
B. caption 出力専用に find_tag_items_by_image_ids を追加する
```

推奨は A です。
今回の同頻度ソートは `tag.name ASC` で足りるため、`tag.id` は必須ではありません。

## 8.6 出力処理中の画面ロック

タグ頻度集計とファイル出力を行うため、出力完了まで LoadingOverlay を表示する。

```text
タイトル: タグを出力中
メッセージ: タグ出現頻度を集計し、caption ファイルを出力しています...
```

## 8.7 出力ファイル仕様

既存仕様を維持。

```text
- 画像と同名 .txt
- 画像と同じディレクトリへ出力
- 既存 .txt は上書き
- UTF-8 without BOM
- 本文末尾に LF を 1 つ付与
- タグ 0 件画像はスキップ
```

既存仕様でも、caption 出力は UTF-8 without BOM、末尾 LF 推奨です。

# 11. Repository 仕様

## 11.1 追加メソッド

```python
def merge_tags_atomic(self, image_file_id: int, tags: list[str]) -> bool:
    ...
```

戻り値。

```text
True:
タグ構成が変更された

False:
追加対象タグがすべて既存で、変更なし
```

## 11.2 処理内容

```text
1. 既存タグを取得する
2. 追加タグと統合する
3. 重複を除去する
4. 変更がなければ False を返す
5. tag_image_link を更新する
6. tag_hash を更新する
7. commit
8. True を返す
```

## 11.3 注意

`replace_tags` を直接使う場合でも、呼び出し前に必ず既存タグと追加タグを統合する。

```text
replace_tags(image_file_id, caption_tags)
```

は禁止。

正しくは。

```text
merged_tags = existing_tags + caption_tags
replace_tags(image_file_id, merged_tags)
replace_tag_hash(image_file_id)
```

# 12. TagCaptionExportService 変更

既存の `TagCaptionExportService` に頻度順ソートを追加する。

## 12.1 追加処理

```text
1. 対象画像全体のタグ一覧を取得する
2. タグごとの出現画像数を集計する
3. 各画像のタグを以下で並べ替える
   - 出現画像数 DESC
   - tag.name ASC
4. ", " で連結して出力する
```

## 12.2 実装イメージ

```python
def build_frequency_map(tags_by_image_id: dict[int, list[str]]) -> dict[str, int]:
    frequency: dict[str, int] = {}

    for tags in tags_by_image_id.values():
        for tag in set(tags):
            frequency[tag] = frequency.get(tag, 0) + 1

    return frequency


def sort_tags_by_frequency(tags: list[str], frequency: dict[str, int]) -> list[str]:
    return sorted(
        tags,
        key=lambda tag: (-frequency.get(tag, 0), tag),
    )
```

# 14. 受け入れ条件

## 14.1 AppHeader.vue

```text
- タグ操作ドロップダウンが表示されること
- プロンプトの読み取りがタグ操作配下へ移動していること
- キャプションタグ読み込みがタグ操作配下に表示されること
- 一括タグ追加がタグ操作配下に表示されること
- 重複タグ構成の検索がタグ操作配下へ移動していること
- selectedCount === 0 の場合、キャプションタグ読み込みと一括タグ追加が disabled になること
```

## 14.2 キャプションタグ読み込み

```text
- 選択画像と同名の .txt を読み込めること
- 既存タグを保持したまま caption タグを追加できること
- "," 区切りで複数タグを読み込めること
- 同名 .txt がない画像はスキップされること
- 空ファイルはスキップされること
- 追加後に tag_hash が更新されること
- 完了後に現在検索条件で再検索されること
```

## 14.3 一括タグ追加

```text
- 一括タグ追加ダイアログが表示されること
- "," 区切りで複数タグを入力できること
- 選択画像すべてへ同一タグを追加できること
- 既存タグは保持されること
- 重複タグは二重登録されないこと
- 追加後に tag_hash が更新されること
- 完了後に現在検索条件で再検索されること
```

## 14.4 タグ出力

```text
- 選択画像内でのタグ出現頻度を集計できること
- 各 caption ファイル内のタグ順が出現頻度 DESC になること
- 同頻度の場合は tag.name ASC になること
- タグ 0 件画像はスキップされること
- 既存 .txt は上書きされること
- UTF-8 without BOM で出力されること
- 出力中は LoadingOverlay で画面ロックされること
```

## 注意点

最大の注意点は、**merge と replace を取り違えないこと**です。

今回の 2 機能はどちらも「追加」であって「置換」ではありません。

```text
キャプションタグ読み込み:
既存タグ + caption タグ

一括タグ追加:
既存タグ + 入力タグ
```

ここを誤ると LoRA 用 caption 調整のつもりが、既存タグをきれいに更地にします。更地にするならせめて不動産登記をしてからにしましょう。
